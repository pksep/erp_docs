## Путь
Склад -> Комплектация сборок на план -> Двойной клик на Изображение или Обозначение сущности в таблице -> В таблице "Детали" с сущностью, кликаем на содержания колонки "Общая потребность на склад".

## Описание
Модальное окно "Принадлежность" предназначено для отображения информации о родительской сущности, для которой изготовляется данный потомок.\
Оно также предоставляет общее количество заказов, связанных с этой родительской сущностью, что позволяет пользователям быстро оценить контекст и статус производства.

### Особенности
- **Интуитивный интерфейс**
  - Модальное окно имеет простой и понятный интерфейс, что облегчает восприятие информации о родительской сущности и связанных заказах.

### Функционал
- **Отображение информации о выбранной сущности**
  - Модальное окно отображает тип сущности и его наименование, а так же обозначение.
- **Отображение информации о родительской сущности**
  - Модальное окно показывает название и артикул родительской сущности, к которой относится данный потомок.
- **Общее количество заказов**
  - Отображает общее количество заказов, связанных с родительской сущностью, что позволяет пользователям быстро оценить объем работы.
- **Закрытие модального окна**
  - Пользователь может закрыть модальное окно, нажав на кнопку "Закрыть" или кликнув вне области модального окна, возвращаясь к предыдущему интерфейсу.

---
## Требования
- **Заголовок модального окна**
  - Заголовок должен отображать текст "Принадлежность" с указанием выбранного потомка.
- **Информация о родительской сущности**
  - Модальное окно должно отображать название, артикул и тип родительской сущности, к которой относится данный потомок.
- **Общее количество заказов**
  - Должно отображаться общее количество заказов, связанных с родительской сущностью.
 **Таблица "Принадлежность"** - Должна содержать следующие столбцы:
  - №
  - Тип
  - Обозначение
  - Наименование
  - Заказы
  - Кол-во (с подколонками ЗП и ЗС)
  - Потребность на 1 шт.
  - Итого по заказам склада
  - Итог (с пометкой "Итого: *Количество потребностей по всем заказам склада*")
  - Описание потомка
- **Отображение данных**
  - Модальное окно должно отображать актуальную информацию о родительской сущности и связанных заказах.
- **Просмотр заказов**
  - Пользователь должен иметь возможность кликнуть на иконку в столбце "Заказы", чтобы открыть модальное окно "Заказы" со списком заказов родительской сущности.
- **Закрытие модального окна**
  - Пользователь должен иметь возможность закрыть модальное окно, нажав на кнопку "Закрыть" или кликнув вне области модального окна.
- **Адаптивность**
  - Тип потомка (Изделие/Сборка/Деталь) должен изменяться в зависимости от типа выбранного потомка.

## Описание
Таблица "Принадлежность" (Table 1) отображает родительские сущности, заказанные на производстве, в которых участвует выбранный потомок, а также количество потребностей по этим заказам.

### Структура
- **№**
  - Порядковый номер в таблице.
- **Тип**
  - Тип родительской сущности (например, Изд., СБ, Д.).
- **Обозначение**
  - Уникальное обозначение родительской сущности.
- **Наименование**
  - Полное название родительской сущности.
- **Заказы**
  - [Иконка](/img/plus.png), при нажатии на которую открывается модальное окно "Заказы" со списком заказов родительской сущности.
- **Кол-во** (Разделен на две подколонки)
  - **ЗП**: Отображает общее количество по заказам покупателей родительской сущности.
  - **ЗС**: Отображает общее количество по заказам склада.
- **Потребность на 1 шт.**
  - Необходимое количество данного потомка для изготовления одной единицы родительской сущности.
- **Итого по заказам склада**
  - Общее количество по заказам склада
- **Итог**
  - Под таблицей подведен итог необходимого количества данного потомка по всем заказам склада с пометкой "Итого: *Количество потребностей по всем заказам склада*".
- **Описание потомка**
  - Над таблицей отображается описание данного потомка, включая его тип (Изделие/Сборка/Деталь), наименование и обозначение.

### Особенности
- **Интуитивный интерфейс**
  - Удобное представление данных в виде таблицы, что облегчает работу с родительскими сущностями и их потребностями.
- **Адаптивность**
  - Тип потомка (Изделие/Сборка/Деталь) меняется в зависимости от типа выбранного потомка.

### Функционал
- **Отображение информации о родительских сущностях**
  - Таблица показывает данные о родительских сущностях, связанных с выбранным потомком, что позволяет пользователям быстро оценить состояние заказов.
- **Просмотр заказов**
  - Пользователь может кликнуть на иконку в столбце "Заказы", чтобы открыть модальное окно с подробным списком заказов для родительской сущности.
- **Итоговое количество потребностей**
  - Под таблицей отображается общее количество потребностей данного потомка по всем заказам склада, что помогает в планировании и управлении запасами.
