## Путь
Склад (/sclad) -> Заказано у поставщиков (/ordersuppliers) -> Кнопка "Создать заказ" -> "Детали", "Сборка", "Изделии"

---
## Описание
Модальное окно "Создание заказа поставщика" предназначено для выбора сущности в зависимости от выбранного типа сущности и создания заказа для производства.\
Оно предоставляет пользователям возможность управлять процессом создания заказа, а также изменять поставщика при необходимости.

### Особенности
- **Выбор сущности**
    - Модальное окно позволяет пользователям выбирать различные сущности, связанные с заказом, в зависимости от выбранного типа сущности.
- **Изменение поставщика**
    - Пользователи могут изменить тип сущности.
- **Интуитивный интерфейс**
    - Простота и ясность интерфейса позволяют пользователям легко ориентироваться и выполнять необходимые действия.

### Функционал
- **Создание заказа**
    - Пользователи могут создать заказ для производства, выбирая соответствующую сущность.

---
## Требования 
- **Заголовок модального окна**
   - Заголовок должен отображать текст "Создание заказа поставщика".
- **Информация о поставщике**
   - Должно отображаться сообщение в формате "Поставщик: *Выбранный тип сущности*".
- **Кнопка "Изменить выбор"**
   - Должна быть доступна для изменения типа поставщика.
- **Таблица сущностей** - Должна содержать следующие столбцы:
    - Чекбокс
    - №
    - Обозначение
    - Наименование
    - На складе
    - Заказано у покупателя
    - Минимальный остаток
    - Заказано на производстве
    - Свое кол-во
    - Заказы
- **Выбор сущности**
   - Пользователь должен иметь возможность отмечать необходимые сущности с помощью чекбоксов.
- **Установка количества**
   - Пользователь должен иметь возможность вводить количество в поле "Свое кол-во" для каждой сущности.
- **Создание заказа**
   - Кнопка "Заказать" должна переносить отмеченные сущности в форму для запуска в производство.
   - При нажатии на кнопку "Заказать" должно открываться модальное окно "Запустить в производство".
   - Если не отмечена ни одна сущность в таблице, должно всплывать уведомление "Сначала выберите объекты для запуска в производство".
- **Изменение поставщика**
   - При нажатии на кнопку "Изменить выбор" должно открываться модальное окно с выбором типа поставщика.
- **Закрытие модального окна**
   - Кнопка "Закрыть" должна закрывать модальное окно.

## Описание
В модальном окне отображается информация о выбранном поставщике в формате "Поставщик: *Выбранный тип сущности*".\
Это позволяет пользователям быстро идентифицировать текущий выбор поставщика.\
Напротив этой информации находится кнопка "Изменить выбор", которая предоставляет возможность изменить тип поставщика.

### Особенности
- **Отображение текущего выбора**
    - Название типа поставщика четко показывает, какой тип поставщика был выбран, что помогает пользователям быстро ориентироваться.
- **Интуитивный интерфейс**
    - Простота и ясность интерфейса делают процесс изменения выбора поставщика легким и понятным.
- **Кнопка "Изменить выбор"** (Button 1)
    - Позволяет пользователям легко инициировать процесс изменения типа поставщика.

### Функционал
- **Изменение типа поставщика**
    - При клике на кнопку "Изменить выбор" открывается модальное окно с выбором типа поставщика, где пользователи могут выбрать новый тип.

## Описание таблицы (Table 1)
Таблица типа сущности отображает список выбранной сущности с возможностью отметить необходимые, установить количество и создать заказ для производства.\
Она предоставляет пользователям удобный интерфейс для управления сущностями, которые будут использоваться в производственных процессах.

### Структура
- **Чекбокс в виде галочки**
    - Отображает ячейки с чекбоксом для отметки сущности, с которыми будут производиться манипуляции.
- **№**
    - Порядковый номер в таблице.
- **Обозначение**
    - Обозначение сущности.
- **Наименование**
    - Наименование сущности.
- **На складе**
    - Отображает количество сущности на складе в свободном остатке.
- **Заказано у покупателя**
    - Общее количество сущности по заказам покупателей.
- **Минимальный остаток**
    - Рекомендуемое минимальное количество остатка, информация берется из карточки сущности.
- **Заказано на производстве**
    - Общее количество заказанных сущностей по заказам склада.
- **Свое кол-во**
    - Поле ввода, позволяющее ввести количество, которое пользователь планирует запустить в производство.
- **Заказы**
    - Иконка, при клике на которую открывается модальное окно "Список заказов" выбранной сущности с информацией о заказах склада и заказов покупателей.
- **Поиск**
    - Расположен под описанными выше колонками и занимает всю их ширину.
    - Позволяет искать детали по "Артикулу" или "Наименованию".
    - Включает возможность просмотра истории запросов поиска.
    - Используется placeholder для подсказки пользователю допустимых значений поиска.

### Особенности
- **Поиск**
    - Внутри таблицы под заголовками столбцов расположены поля поиска с возможностью поиска по "Обозначению" или "Наименованию".
- **Отображение количества отмеченных сущностей**
    - Над таблицей выводится информация "Выбрано: *количество отмеченных сущностей*", отображающая количество отмеченных сущностей в таблице при помощи чекбокса.

### Функционал
- **Закрытие модального окна** (Button 2)
    - Кнопка "Закрыть" закрывает модальное окно.
- **Кнопка "Заказать"** (Button 3)
    - Кнопка "Заказать" позволяет перенести отмеченные сущности в форму для запуска в производство.
    - При нажатии открывается модальное окно "Запустить в производство", в котором формируется заказ, и есть возможность свериться с отмеченными сущностями и количеством, указанным в столбце "Свое кол-во", чтобы запустить производство.
    - Если не отмечена ни одна сущность в таблице, всплывает уведомление "Сначала выберите объекты для запуска в производство", что помогает избежать ошибок при запуске.
