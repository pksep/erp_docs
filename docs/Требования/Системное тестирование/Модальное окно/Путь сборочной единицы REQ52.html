Путь сборочной единицы REQ52

Путь​

Склад (/sclad) -> Комплектация сборок на план (/completcbed) -> клик на иконку в колонке операции.

Описание​

Модальное окно "Путь сборочной единицы" предназначено для отображения информации о заказах данной сущности, краткой информации о комплектации, а также этапов операций из технологического процесса изготовления сущности.
Оно позволяет пользователям отслеживать статус на каждом этапе, количество выполненных задач и оставшихся для завершения, а также управлять отметками о выполнении.

Особенности​

  • Интуитивный интерфейс
    • Модальное окно имеет простой и понятный интерфейс, что облегчает восприятие информации о заказах и этапах операций.
  • Динамическое обновление
    • Информация в окне обновляется в реальном времени, что обеспечивает актуальность данных о статусах операций.
  • Печать информации
    • Возможность распечатать содержание модального окна для удобства работы с документами.

Функционал​

  • Отображение информации о заказах
    • Модальное окно показывает список заказов, связанных с выбранной сущностью, включая ключевые характеристики.
  • Краткая информация о комплектации
    • Отображает основные данные о комплектации сущности, что позволяет пользователям быстро оценить состояние выполнения заказа.
  • Этапы операций - Информация о каждом этапе технологического процесса изготовления сущности, включая:
    • Статус на каждом этапе (В процессе/Готово).
    • Количество выполненных задач.
    • Количество оставшихся задач для завершения.
  • Проставление отметки о выполнении:
    • Пользователь может проставить отметку о выполнении на различных этапах операций, что позволяет отслеживать прогресс выполнения.
  • Отмена отметки о выполнении:
    • Возможность отправить отметку о выполнении в архив, что позволяет отменить её.
  • Печать содержания:
    • Кнопка для отправки содержания модального окна на предпросмотр печати, позволяя пользователю распечатать информацию о заказах и этапах операций.

Описание таблицы с заказами (Table 1)​

Таблица с информацией о заказах предназначена для отображения данных о заказах склада. Она предоставляет пользователям возможность быстро оценить состояние заказов, включая сроки отгрузки, количество заказанных сущностей и уровень готовности.

Структура таблицы​

  • № Заказа
    • Уникальный номер заказа, позволяющий идентифицировать каждый заказ.
  • Срок отгрузки
    • Дата планируемой отгрузки, указанная в заказе, что позволяет пользователям отслеживать сроки выполнения.
  • Кол-во, шт.
    • Количество заказанных сущностей в заказе, что помогает оценить объем работы.
  • Готовность %
    • Процент готовности заказа, показывающий, насколько близок заказ к выполнению.
  • Готовность к сборке
    • Статус готовности к сборке сущности в заказе, который может быть "Готово" или "В процессе".

Особенности​

  • Интуитивный интерфейс
    • Таблица организована в логическом порядке, что облегчает восприятие информации о заказах.
  • Структурированность
    • Четкое разделение информации по столбцам позволяет быстро находить необходимые данные.

Функционал​

  • Отображение информации о заказах
    • Таблица показывает все необходимые данные о заказах, что позволяет пользователям быстро оценить состояние выполнения.
  • Фильтрация и сортировка
    • Пользователи могут фильтровать и сортировать данные по различным столбцам, что упрощает поиск нужной информации.
  • Обновление данных
    • Возможность обновления данных в реальном времени, чтобы пользователи всегда имели доступ к актуальной информации о заказах.

Описание таблицы "Комплектация" (Table 2)​

Таблица "Комплектация" предназначена для отображения потомков первой линии и содержит информацию о потребностях каждого потомка, количестве свободного остатка на складе и общем дефиците. Она позволяет пользователям быстро оценить состояние запасов и потребностей для успешной комплектации родительской сущности.

Структура​

  • Обозначение
    • Уникальное обозначение потомка, позволяющее идентифицировать его.
  • Наименование
    • Полное название потомка, что помогает пользователям быстро понять, о каком элементе идет речь.
  • Ед.
    • Единица измерения потомка (например, шт., кг, м и т.д.).
  • Кол-во на 1 СБ
    • Необходимое количество единиц потомка для изготовления одной родительской сущности.
  • Потребность
    • Общее необходимое количество потомка для выполнения всех заказов родительской сущности.
  • Кол-во всего, шт.
    • Количество потомка, доступного на складе.
  • Дефицит всего, шт.
    • Общий дефицит потомка по всем заказам на складе, показывающий, сколько единиц не хватает для выполнения заказов.

Особенности​

  • Интуитивный интерфейс
    • Таблица организована в логическом порядке, что облегчает восприятие информации о потребностях и запасах.
  • Структурированность
    • Четкое разделение информации по столбцам позволяет быстро находить необходимые данные.

Функционал​

  • Отображение информации о потомках
    • Таблица показывает все необходимые данные о потомках первой линии, что позволяет пользователям быстро оценить состояние запасов и потребностей.

Описание таблицы "Детали операции" (Table 3)​

Таблица "Детали операций" отображает этапы операций, через которые должна пройти родительская сущность для её изготовления.
Она предоставляет пользователям возможность проставить отметки о выполнении на различных этапах операций, а также отслеживать статус выполнения на каждой операции.
Этапы операций берутся из технологического процесса, указанного в карточке сущности.

Структура​

  • №
    • Порядковый номер в таблице.
  • Операции
    • Название операции из технологического процесса данной сущности.
  • Норма времени на 1 шт., час
    • Норма времени для прохождения данного этапа операции в часах (данные берутся из карточки сущности).
  • Норма времени, итого
    • Итоговая норма времени для прохождения данного этапа операции (данные берутся из карточки сущности).
  • Выполнено в н/ч
    • Фактическое затраченное время детали на данной операции (данные берутся из карточки сущности).
  • Статус
    • Статус детали на разных этапах операций (В процессе/Готово).
  • Сделано, шт.
    • Количество сущностей, которые прошли данный этап операции.
  • Осталось сделать, шт.
    • Количество сущностей заказанных складом, которые еще должны пройти данный этап операции.
  • Дата исполнения
    • Дата проставления отметки.
    • Если отметок было несколько, ячейка делится на количество отметок.
  • Кол-во сделано, шт.
    • Количество деталей, прошедших через данный этап операции.
    • Если отметок было несколько, ячейка делится на количество отметок.
  • Исполнитель
    • ФИО сотрудника, который проставлял отметку о выполнении.
    • Если отметок было несколько, ячейка делится на количество отметок.
  • Примечание
    • Иконка, при клике на которую открывается модальное окно с информацией, оставленной при проставлении отметки о выполнении.
    • Если отметок было несколько, ячейка делится на количество отметок.
  • Обновить отметку
    • Иконка, при клике на которую открывается модальное окно "Отметка о выполнении" с заполненными данными, внесенными при проставлении отметки о выполнении.
    • Если отметок было несколько, ячейка делится на количество отметок.

Особенности​

  • Динамическое обновление
    • Таблица автоматически обновляется с учетом отмененных отметок, что обеспечивает актуальность данных.
  • Строгое соблюдение технологического процесса
    • Все этапы операций идут в строгом соответствии с технологическим процессом данной сущности, что позволяет избежать ошибок в производстве.

Функционал​

  • Проставление отметок о выполнении
    • Пользователь может проставить отметку о выполнении на различных этапах операций, что позволяет отслеживать прогресс выполнения.
  • Добавление отметки для выбранной операции (Button 2)
    • При клике на ячейки от "№" до "Осталось сделать, шт." таблицы "Детали операции" появляется скрытая кнопка "Добавить отметку для выбранной операции", открывающая модальное окно "Отметка о выполнении" для создания новой отметки.
  • Архивирование отметок (Button 3)
    • При клике на ячейки от "Дата исполнения" до "Обновить отметку" таблицы "Детали операции" появляется скрытая кнопка "В архив", позволяющая отправить выбранную отметку о выполнении в архив.
  • Печать содержания (Button 1)
    • Кнопка "Печать" позволяет распечатать содержание таблицы с этапами операций.
