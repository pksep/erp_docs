## Путь
Склад -> Комплектация сборок на план -> Двойной клик на Изображение или Обозначение сущности в таблице -> В таблице "Детали" с сущностью, кликаем на иконку в колонке "Металлообработка: Заказы".

## Описание
Модальное окно "Список заказов" предназначено для отображения заказов покупателей и заказов склада.\
Оно предоставляет краткую информацию о каждом заказе, а также общее количество заказов покупателей и складских заказов, включая количество оставшихся заказов для выполнения заказов склада.\
В заголовке окна отображается тип сущности, его наименование и обозначение.

### Особенности
- **Краткая информация**
    - Отображение краткой информации о каждом заказе, что позволяет быстро оценить состояние заказов.
- **Отображение заказов**
    - Заказы покупателей и заказы склада отображаются в отдельных таблицах, чтобы избежать путаницы.
- **Заголовок модального окна**
    - В заголовке отображается тип сущности, его наименование и обозначение, что помогает пользователю быстро идентифицировать контекст информации.

### Функционал
- **Общее количество заказов**
    - Отображение общего количества заказов покупателей и складских заказов внизу каждой таблицы.

---
## Требования
- **Заголовок модального окна**
    - Заголовок должен отображать тип сущности, его наименование и обозначение.
- **Таблица "Заказы покупателей"** - Должна отображать следующие столбцы:
    - № Заказа.
    - Дата заказа.
    - Планируемая дата отгрузки (с возможностью сортировки по этому столбцу).
    - Кол-во изделий на заказ.
- Под таблицей должно отображаться:
    - Общее количество заказанных единиц в формате: "Всего: *Количество заказанной сущности*".
- **Таблица "Заказы склада"** - Должна отображать следующие столбцы:
    - № Заказа.
    - Дата заказа.
    - Планируемая дата отгрузки.
    - Кол-во изделий на заказ.
    - Отгружено по заказу.
- Под таблицей должно отображаться:
    - Итого по заказам: *Количество заказанной сущности*.
    - Осталось сделать: *Общее количество по заказам склада с учетом уже отгруженных единиц*.
- **Закрытие модального окна**
    - Пользователь должен иметь возможность закрыть модальное окно, нажав на кнопку "Закрыть" или кликнув вне области модального окна.

## Описание
Информация о заказах представлена в виде двух таблиц: "Заказы покупателей" и "Заказы склада".\
Каждая таблица содержит ключевую информацию о заказах, связанных с выбранной сущностью, и позволяет пользователям быстро оценить состояние заказов и их выполнение.

### Структура таблицы "Заказы покупателей" (Table 1)
- **№ Заказа**
    - Номер заказа, присвоенный при его формировании.
- **Дата заказа**
    - Дата оформления заказа.
- **Планируемая дата отгрузки**
    - Дата, когда планируется отгрузка заказа (таблица сортируется по этому столбцу от самых ранних).
- **Кол-во Изделий на Заказ**
    - Количество заказанных единиц сущности в заказе.
- **Итог**
    - Под таблицей отображается общее количество заказанной сущности в заказах покупателей в формате: "Всего: *Количество заказанной сущности*".

### Структура таблицы "Заказы склада" (Table 2)
- **№ Заказа**
    - Номер заказа, присвоенный при его формировании.
- **Дата заказа**
    - Дата оформления заказа.
- **Планируемая дата отгрузки**
    - Дата, когда планируется отгрузка заказа (таблица сортируется по этому столбцу от самых ранних).
- **Кол-во Изделий на Заказ**
    - Количество заказанных единиц сущности в заказе.
- **Отгружено по заказу**
    - Количество уже отгруженных единиц сущности по заказу склада.
- **Итог**
    - Под таблицей отображается общее количество заказанной сущности в заказах склада в формате: "Итого по заказам: *Количество заказанной сущности*".
    - Также отображается строка: "Осталось сделать: *Общее количество по заказам склада с учетом уже отгруженных сущностей по заказу*".

### Особенности
- **Разделение информации**
    - Четкое разделение информации о заказах покупателей и заказах склада для удобства восприятия.
- **Сортировка**
    - Возможность сортировки таблицы "Заказы покупателей" по планируемой дате отгрузки, что позволяет быстро находить наиболее срочные заказы.
- **Итоговая информация**
    - Отображение итогов по заказам в каждой таблице, что позволяет пользователям быстро оценить объем заказов и оставшиеся обязательства.

### Функционал
- **Отображение заказов**
    - Каждая таблица отображает соответствующие заказы с ключевой информацией, что позволяет пользователям быстро оценить состояние заказов.
- **Сортировка**
    - Пользователь может сортировать заказы покупателей по планируемой дате отгрузки.
- **Итоговая информация**
    - Под каждой таблицей отображается итоговая информация о количестве заказанных сущностей, что помогает пользователям быстро оценить общую нагрузку.
