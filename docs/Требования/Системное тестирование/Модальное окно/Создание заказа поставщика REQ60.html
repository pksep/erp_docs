Создание заказа поставщика REQ60

Путь​

Склад (/sclad) -> Заказано у поставщиков (/ordersuppliers) -> Кнопка "Создать заказ" -> "Поставщик"

Описание​

Модальное окно "Создание заказа поставщика" предназначено для создания заказа у поставщика.
Оно выполнено в виде модального окна с формой для заполнения данных, что позволяет пользователям удобно выбирать заказываемый продукт из списка доступных товаров и указывать необходимые параметры заказа, такие как цена и количество.

Особенности​

  • Выбор продукта
    • Модальное окно предоставляет возможность выбора заказываемого продукта из списка, что упрощает процесс создания заказа.
  • Интуитивный интерфейс
    • Простота и ясность интерфейса делают процесс заполнения формы легким и понятным для пользователей.
  • Динамическое обновление
    • Список доступных продуктов обновляется в зависимости от выбранного поставщика, что обеспечивает актуальность информации.

Функционал​

  • Создание заказа
    • Пользователи могут заполнять форму с указанием выбранного продукта, его цены и количества, что позволяет создать заказ у поставщика.
    • После заполнения всех необходимых полей пользователи могут создать заказ.
  • Отмена заказа
    • Пользователи могут отменить создание заказа, если передумали, что обеспечивает гибкость в управлении процессом.
  • Валидация данных
    • Модальное окно должно проверять корректность введенных данных (например, наличие обязательных полей), чтобы избежать ошибок при создании заказа.

Требования​

  • Заголовок модального окна
    • Заголовок должен отображать текст "Создание заказа поставщика".
  • Форма для заполнения - Должна включать следующие поля:
    • Поставщик: Поле для ввода имени поставщика с кнопкой "Выбрать" для выбора из списка.
    • № Счета: Поле ввода для номера счета поставщика.
    • Сумма счета, руб: Поле ввода для общей суммы заказа.
    • Планируемая дата прихода: Поле для указания даты поступления заказа с календарем для выбора.
    • НДС %: Поле ввода для указания процента НДС (по умолчанию 20%).
    • Кнопка "Загрузить счет": Для загрузки файла с компьютера.
  • Таблица "Дефицитный материал" - Должна содержать столбцы:
    • Категория
    • Тип
    • Подтип
    • Наименование
  • Таблица "Список поставляемого поставщиком"
    • Должна отображать категории продуктов, привязанные к карточке выбранного поставщика.
  • Таблица "Выбранные позиции" - Должна содержать следующие столбцы:
    • Обозначение
    • Наименование
    • Единица измерения (ЕИ)
    • Количество
    • Сумма, руб (за шт.)
    • Примечание
    • Тип
  • Создание заказа
    • Пользователь должен иметь возможность заполнять форму с указанием выбранного продукта, его цены и количества.
    • После заполнения всех необходимых полей пользователь должен иметь возможность создать заказ.
  • Отмена заказа
    • Пользователь должен иметь возможность отменить создание заказа.
  • Валидация данных
    • Модальное окно должно проверять корректность введенных данных (например, наличие обязательных полей).
  • Добавление позиций
    • Пользователь должен иметь возможность добавлять позиции в таблицу "Выбранные позиции" с помощью кнопки "Добавить к поставщику".
  • Редактирование позиций
    • Пользователь должен иметь возможность редактировать количество и цену в таблице "Выбранные позиции".
  • Удаление позиций
    • Кнопка "Удалить выбранное" должна удалять выделенные позиции из таблицы "Выбранные позиции".
  • Очистка списка
    • Кнопка "Очистить" должна полностью очищать список выбранных позиций.
  • Создание заказа
    • Кнопка "Заказать" должна создавать заказ с выбранными позициями, при этом должны быть выполнены следующие условия:
      • Должен быть выбран поставщик.
      • Должен быть заполнен номер счета.
      • Должна быть выбрана хотя бы одна позиция для заказа.
    • Если условия не выполнены, должно всплывать уведомление с просьбой заполнить необходимые поля.
  • Уведомление об успешности операции
    • При успешном создании заказа модальное окно должно закрываться, и пользователю должно отображаться уведомление об успешности операции.
  • Закрытие модального окна
    • Кнопка "Закрыть" должна закрывать модальное окно.

Описание​

При открытии модального окна пользователю представляется форма для заполнения, предназначенная для создания заказа у поставщика.
Форма включает в себя несколько полей, которые позволяют пользователю ввести необходимую информацию о заказе, включая данные о поставщике, счете и условиях заказа.

Особенности​

  • Поле "Поставщик:" (Input 1)
    • Пользователь может вручную ввести имя поставщика или выбрать его из списка, нажав кнопку "Выбрать" (Button 1), которая открывает список поставщиков, внесенных в базу данных.
  • Поле "№ Счета:" (Input 3)
    • Поле ввода для внесения номера счета поставщика, что позволяет точно идентифицировать заказ.
  • Поле "Сумма счета, руб:" (Input 4)
    • Поле ввода для указания общей суммы заказа, что помогает в финансовом учете.
  • Поле "Планируемая дата прихода:" (Input 2)
    • Пользователь может указать дату поступления заказа, при этом по умолчанию установлена сегодняшняя дата, что упрощает процесс заполнения.
    • При наведении курсора всплывает календарь для удобства выбора планируемой даты поступления заказа.
  • Поле "НДС %:" (Input 5)
    • Поле ввода для внесения процента по НДС, по умолчанию установленное значение 20%, что соответствует стандартной ставке.
  • Кнопка "Загрузить счет": (Button 3)
    • Позволяет пользователю загрузить файл с компьютера и прикрепить его к заказу, что обеспечивает удобство в управлении документами.
  • Кнопка "Изменить выбор" (Button 2)
    • Позволяет пользователям легко инициировать процесс изменения типа поставщика.

Функционал​

  • Ввод данных о поставщике
    • Пользователь может вводить данные о поставщике вручную или выбирать из списка, что упрощает процесс создания заказа.
  • Заполнение информации о счете
    • Пользователь может ввести номер счета и общую сумму, что позволяет точно указать условия заказа.
  • Указание даты поступления
    • Возможность указать планируемую дату прихода заказа, что помогает в планировании логистики.
  • Ввод НДС
    • Пользователь может указать процент НДС, что важно для правильного расчета стоимости заказа.
  • Загрузка документов
    • Кнопка "Загрузить счет" позволяет прикрепить необходимые документы к заказу, что упрощает процесс учета и хранения информации.

Описание таблицы "Дефицитный материал" (Table 1)​

Таблица "Дефицитный материал" предназначена для быстрого внесения заказываемого продукта у поставщика.
Данные подтягиваются из внесенных баз данных в системе, что обеспечивает актуальность информации и упрощает процесс заказа необходимых материалов и инструментов.

Структура​

  • Категория - Указывает доступные категории для заказа:
    • Все: Отображает все категории в таблице.
    • Материалы: Отображает только материалы, внесенные в базу данных материалов (установлено по умолчанию).
    • Инструмент: Отображает только инструменты, внесенные в базу данных инструментов.
    • Оборудование: Отображает только оборудование, внесенное в базу данных оборудования.
    • Техника и Инвентарь: Отображает только данные из базы данных техники и инвентаря.
  • Тип
    • Отображает типы выбранной категории согласно базам данных.
  • Подтип
    • Отображает подтипы выбранного типа.
  • Наименование
    • Отображает наименования сущностей, находящихся в выбранном подтипе. Под заголовком столбца есть поле для поиска по наименованию с подсказкой "Поиск по Наименованиям".

Особенности​

  • Фильтрация по категориям
    • При клике на каждую категорию остальные столбцы таблицы фильтруются согласно выбранной категории, что позволяет пользователям быстро находить нужные материалы или инструменты.
  • Поиск по наименованию
    • Поле для поиска по наименованию позволяет пользователям быстро находить конкретные сущности в таблице, улучшая удобство работы с данными.

Функционал​

  • Добавление к поставщику (Button 4)
    • Кнопка "Добавить к поставщику" позволяет отправить выбранное наименование сущности к списку выбранных позиций для дальнейшего заказа, что упрощает процесс формирования заказа.
  • Динамическое обновление
    • Таблица автоматически обновляется в зависимости от выбранной категории, что обеспечивает актуальность информации и удобство работы.
  • Интерактивность
    • Пользователи могут легко взаимодействовать с таблицей, выбирая категории и подкатегории, что делает процесс заказа более удобным.

Описание таблицы "Список поставляемого поставщиком" (Table 2)​

Таблица "Список поставляемого поставщиком" представляет собой список категорий продуктов, которые привязаны к карточке выбранного поставщика.
Эта таблица позволяет пользователям быстро просматривать доступные продукты и их категории, что упрощает процесс выбора и заказа товаров у поставщика.

Структура​

  • Категория - Указывает доступные категории для заказа:
    • Все: Отображает все категории в таблице.
    • Материалы: Отображает только материалы, внесенные в базу данных материалов (установлено по умолчанию).
    • Инструмент: Отображает только инструменты, внесенные в базу данных инструментов.
    • Оборудование: Отображает только оборудование, внесенное в базу данных оборудования.
    • Техника и Инвентарь: Отображает только данные из базы данных техники и инвентаря.
  • Тип
    • Отображает типы выбранной категории согласно базам данных.
  • Подтип
    • Отображает подтипы выбранного типа.
  • Наименование
    • Отображает наименования сущностей, находящихся в выбранном подтипе. Под заголовком столбца есть поле для поиска по наименованию с подсказкой "Поиск по Наименованиям".

Особенности​

  • Категории продуктов
    • Таблица отображает различные категории продуктов, которые предлагает выбранный поставщик, что помогает пользователям ориентироваться в ассортименте.
  • Привязка к карточке поставщика
    • Данные в таблице автоматически обновляются в зависимости от выбранного поставщика, обеспечивая актуальность информации.
  • Фильтрация по категориям
    • При клике на каждую категорию остальные столбцы таблицы фильтруются согласно выбранной категории, что позволяет пользователям быстро находить нужные материалы или инструменты.
  • Поиск по наименованию
    • Поле для поиска по наименованию позволяет пользователям быстро находить конкретные сущности в таблице, улучшая удобство работы с данными.

Функционал​

  • Просмотр категорий
    • Пользователи могут просматривать список категорий продуктов, доступных у выбранного поставщика, что помогает в принятии решений о заказе.
  • Фильтрация и сортировка
    • Возможность фильтрации при помощи выбора категорий и поиска внутри таблиц, что упрощает поиск нужных продуктов.
  • Интерактивность
    • Пользователи могут кликать на категории для получения дополнительной информации о продуктах, что улучшает взаимодействие с таблицей.
  • Добавление к поставщику (Button 5)
    • Кнопка "Добавить к поставщику" позволяет отправить выбранное наименование сущности к списку выбранных позиций для дальнейшего заказа, что упрощает процесс формирования заказа.
  • Динамическое обновление
    • Таблица автоматически обновляется в зависимости от выбранной категории, что обеспечивает актуальность информации и удобство работы.

Описание таблицы "Выбранные позиции" (Table 3)​

Таблица "Выбранные позиции" отображает все выбранные позиции из предыдущих таблиц, добавленные с помощью кнопки "Добавить к поставщику".
Это позволяет пользователям свериться с итоговым заказом, при необходимости удалить отдельные позиции или очистить полностью список таблицы перед окончательным оформлением заказа.

Структура​

  • Обозначение
    • Отображает обозначение перенесенной сущности.
  • Наименование
    • Наименование сущности.
  • ЕИ
    • Единица измерения (ЕИ) для каждой сущности.
  • Кол-во (Input 6)
    • Поле ввода с указанием количества заказываемых единиц (по умолчанию значение равно 1).
  • Сумма, руб (за шт.) (Input 7)
    • Поле ввода, где можно указать цену за одну единицу.
  • Примечание (Input 8)
    • Поле ввода для добавления примечания к выбранной сущности.
  • Тип
    • Отображает выбранную категорию сущности.

Особенности​

  • Сверка итогового заказа
    • Пользователи могут легко просмотреть все выбранные позиции и убедиться в правильности заказа перед его оформлением.
  • Интерактивные поля ввода
    • Поля для ввода количества и цены позволяют пользователям настраивать заказ в соответствии с их потребностями.
  • Удобное управление списком
    • Возможность удаления отдельных позиций или очистки всего списка упрощает процесс редактирования заказа.

Функционал​

  • Очистка списка (Button 7)
    • Кнопка "Очистить" позволяет полностью удалить все позиции из таблицы, что удобно для начала нового заказа.
  • Удаление выбранного (Button 6)
    • Кнопка "Удалить выбранное" удаляет выделенную сущность из таблицы, что позволяет пользователям управлять списком заказов.
  • Создание заказа (Button 9)
    • Кнопка "Заказать" создает заказ с выбранными позициями. Перед созданием заказа необходимо:
      • Выбрать поставщика.
      • Заполнить номер счета.
      • Выбрать хотя бы одну позицию для заказа.
    • Если какие-либо из этих условий не выполнены, всплывает уведомление с просьбой заполнить необходимые поля.
    • Имеет фиксированное расположение внизу экрана.
  • Уведомление об успешности операции
    • При успешном создании заказа модальное окно закрывается, и пользователю отображается уведомление об успешности операции.
  • Закрытие модального окна (Button 8)
    • Кнопка "Закрыть" закрывает модальное окно, позволяя пользователю вернуться к предыдущему интерфейсу.
    • Имеет фиксированное расположение внизу экрана.
  • Примечание (textarea)
    • Поле ввода для примечания ко всему заказу.
