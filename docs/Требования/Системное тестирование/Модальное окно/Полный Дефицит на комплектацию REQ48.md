## Путь 
Склад (/sclad) -> Комплектация сборок на план (/completcbed) -> клик на иконку в колонке дефициты -> В Кратком дефиците нажимаем на кнопку "Сформировать полный дефицит на комплектацию"

## Описание
Модальное окно "Полный дефицит на комплектацию" предназначено для отображения полной информации о дефиците потомков для комплектации родительской сущности.\
Оно включает таблицу с полным списком потомков из полной спецификации к сущности и отображает информацию о дефиците на комплектацию родительской сущности всех потомков, что позволяет пользователям быстро оценить состояние комплектации.

### Особенности
- **Интуитивный интерфейс**
    - Модальное окно имеет простой и понятный интерфейс, что облегчает восприятие информации о дефиците.
- **Печать информации**
    - Возможность вывода всей информации из модального окна на печать для удобства работы с документами.

### Функционал
- **Отображение информации о родителе**
    - Таблица с полной информацией о родительской сущности, включая ключевые характеристики, такие как название, обозначение и другие важные данные.
- **Отображение дефицитов потомков**
    - Таблица с дефицитами потомков для комплектации родителя, показывающая информацию о каждом недостающем потомке, включая наименование, количество и статус.
- **Печать**
    - Пользователь должен иметь возможность распечатать содержание таблицы.
- **Закрытие модального окна**
    - Закрытие модального окна путем нажатия на специальную кнопку или кликом вне области модального окна, возвращая пользователя к предыдущему интерфейсу.

## Требования
- **Заголовок модального окна**
   - Заголовок должен отображать текст "Краткий дефицит на комплектацию" с указанием родительской сущности.
- **Таблица "Комплектация"**
   - Должна отображать информацию о родительской сущности и дефицитах потомков, включая все необходимые столбцы:
     - Принадлежность (Наименование и Артикул).
     - Обозначение.
     - Наименование.
     - Единица измерения (Ед).
     - Количество на единицу (Кол-во на ед.).
     - Потребность.
     - Собрано наборов родителя.
     - Общая потребность на склад.
     - Свободное количество.
     - Общее количество.
     - Дефицит.
     - Количество в наборах.
     - Место хранения.
     - Статус.
     - Заказано.
     - Операция.
     - Процент готовности (% Готовности).
- **Динамическое обновление**
   - Модальное окно должно обновляться в реальном времени при изменении данных о дефиците потомков.
- **Просмотр производственных процессов**
   - При клике на иконку операции должно открываться модальное окно с информацией о производственном пути потомка.
- **Печать информации**
   - Пользователь должен иметь возможность распечатать содержание модального окна и удалить не нужные строки из таблицы "Комплектация" для печати.
   - Пользователь должен иметь возможность распечатать только потомков с дефицитом.
- **Закрытие модального окна**
   - Пользователь должен иметь возможность закрыть модальное окно, нажав на кнопку "Закрыть" или кликнув вне области модального окна.

## Описание
Таблица "Информация о родителе" отображает ключевые данные о родительской сущности.\
Она предоставляет пользователям визуальную и текстовую информацию о родительской сущности, а также необходимое количество по заказам склада и ближайшую дату планируемой отгрузки по заказам покупателей.

### Структура
- **Изображение**
    - Визуальное представление родительского изделия, позволяющее пользователям быстро идентифицировать его.
- **Обозначение**
    - Уникальное обозначение сущности.
    - При наведении курсора, шрифт текста становится жирным, а при клике текст копируется. 
- **Наименование**
    - Полное название родительской сущности.
    - При наведении курсора, шрифт текста становится жирным, а при клике текст копируется. 
- **Дата отгрузки**
    - Дата планируемой отгрузки заказов покупателей и заказов склада.
    - Если есть заказы покупателей, берется самая ранняя дата, в ином случае - самая ранняя из заказов склада.
- **Необходимое кол-во (с учетом)**
    - Оставшееся количество родительской сущности для изготовления по заказам склада.

### Особенности
- **Интуитивный интерфейс**
    - Таблица организована в логическом порядке, что облегчает восприятие информации.
- **Визуальная идентификация**
    - Наличие изображения позволяет быстро идентифицировать родительской сущности.

### Функционал
- **Отображение информации**
    - Таблица показывает ключевые данные о родительской сущности и оставшееся количество для изготовления по заказам склада.

## Описание таблицы "Комплектация"
Таблица "Комплектация" отображает дефицит всех потомков и их подчиненных для комплектации родительской сущности.\
Она предоставляет пользователям информацию о необходимых компонентах, их статусе и потребностях для успешной комплектации родительской сущности.

### Структура таблицы
- **Принадлежность** (разделена на 2 подколонки):
    - **Наименование**: Название родительской сущности или потомка, к которому относится другой потомок или его подчиненный.
    - **Артикул**: Артикул родительской сущности.
- **Обозначение**
    - Уникальный идентификатор потомка.
    - При наведении курсора, шрифт текста становится жирным, а при клике текст копируется, если потомок из категории "Сборки". 
    - При двойном клике открывается модальное окно "Краткий дефицит на комплектацию", если потомок из категории "Сборки".
- **Наименование**
    - Полное название потомка.
    - При наведении курсора, шрифт текста становится жирным, а при клике текст копируется. 
    - При двойном клике открывается модальное окно "Краткая Информация".
- **Ед**
    - Единица измерения потомка.
- **Кол-во на ед.**
    - Количество необходимых потомков для комплектации одной родительской сущности.
- **Потребность**
    - Количество потомков, требуемое для завершения изготовления оставшегося количества родительской сущности.
- **Собрано наборов родителя**
    - Количество скомплектованных наборов родительской сущности.
    - При нажатии открывается страница "Скомплектованные наборы" с выбранным набором родительской сущности в новой вкладке.
- **Общая потребность на склад**
    - Общая потребность потомков по всем заказам на складе.
- **Склад** (разделен на 5 подколонок):
    - **Свободное кол-во**: Свободное количество потомков на складе.
    - **Кол-во**: Общее количество потомков на складе.
    - **Дефицит**: Дефицит потомков на общую потребность склада с учетом свободного количества.
    - **В наборах**: Количество потомков во всех скомплектованных наборах.
    - **Место**: Адрес хранения потомка на складе.
- **Статус**
    - Указывает, запущен ли потомок в производство (Заказано/Не заказано).
- **Заказано**
    - Количество заказанных потомков на производстве.
- **Операция**
    - [Иконка](/img/plus.png), при клике открывает модальное окно в зависимости от типа потомка:
    - Для сборочных единиц: "Путь сборочной единицы".
    - Для деталей: "Производственный путь детали", где отображаются необходимые операции потомка на пути производства и его статус на разных этапах.
- **% Готовности**
    - Отображает общий процент готовности потомка на всех этапах операций.

### Особенности
- **Интуитивный интерфейс**
    - Удобное представление данных в виде таблицы, что облегчает работу с дефицитом компонентов.
- **Модальные окна**
    - Возможность открытия модальных окон для просмотра деталей производственных процессов, что улучшает понимание статуса и потребностей.
- **Категории**
    - Таблица делится на соответствующие категории потомков (Сборки, Детали, Покупные Детали), если они имеются, для удобства поиска сущностей и наглядного разделения.

### Функционал
- **Отображение информации о дефиците**
    - Таблица отображает все необходимые данные о потомках и их подчиненных, что позволяет пользователям быстро оценить состояние комплектации.
- **Просмотр производственных процессов**
    - Пользователь может кликнуть на иконку операции, чтобы получить детальную информацию о производственном пути потомка.
- **Анализ потребностей**
    - Таблица позволяет пользователям анализировать потребности в компонентах и их статус на складе, что помогает в планировании и управлении запасами.

## Компоненты управления
Компоненты управления модальным окном обеспечивают пользователям возможность взаимодействовать с модальным окном "Комплектация".\
Они включают кнопки для формирования полного дефицита, закрытия окна и вывода информации на печать.

### Особенности
- **Ясные обозначения**
    - Названия кнопок четко указывают на их функциональность, что облегчает понимание их назначения.
- **Удобство использования**
    - Компоненты управления обеспечивают простоту и удобство взаимодействия с модальным окном.

### Функционал
- **Кнопка "Закрыть"** (Button 1)
    - **Функция**: Позволяет пользователю закрыть модальное окно и вернуться к предыдущему интерфейсу.
- **Кнопка "Вывести поля для печати"** (Button 2)
    - **Функция**: Позволяет отметить строки таблицы для печати. При нажатии перед первым столбцом появляется колонка "Выбрать для печати" с чекбоксами, что позволяет удалить ненужные для печати строки.
- **Кнопка "Печать дефицитов"** (Button 3)
    - **Функция**: Отправляет на предпросмотр печати сущности с дефицитом в потребностях родительской сущности.
