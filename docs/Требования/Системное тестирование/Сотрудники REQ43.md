## Путь
Настройки (/settings) -> Сотрудники (/employee)

## Описание
Страница "Сотрудники" предназначена для подробного отображения сведений о каждом сотруднике.\
Она предоставляет пользователям возможность редактировать персональные данные, отправлять учетные записи сотрудников в архив, а также добавлять новых пользователей в систему.

### Особенности
- **Подробная информация**
    - Страница отображает полные сведения о сотрудниках, включая их личные данные, должности и контактную информацию.
- **Интуитивный интерфейс**
    - Удобный и понятный интерфейс, который позволяет легко находить и редактировать информацию о сотрудниках.
- **Управление учетными записями**
    - Возможность архивирования учетных записей сотрудников, что позволяет поддерживать актуальность данных в системе.
- **Добавление новых пользователей**
    - Функция добавления новых сотрудников в систему с минимальными усилиями.

### Функционал
- **Отображение сведений о сотруднике**
  - Полное отображение персональных данных, включая ФИО, должность, контактные данные и другие важные сведения.
- **Редактирование данных**
  - Возможность редактировать персональные данные сотрудника.
  - Сохранение изменений с уведомлением об успешном обновлении данных.
- **Архивирование учетной записи**
  - Кнопка для отправки учетной записи сотрудника в архив, что позволяет скрыть неактивные учетные записи из общего списка.
  - Подтверждение действия перед архивированием для предотвращения случайных ошибок.
- **Добавление новых пользователей**
  - Форма для ввода данных нового сотрудника, включая необходимые поля для заполнения.
  - Возможность назначения должности и других параметров при добавлении нового пользователя.
  - Уведомление об успешном добавлении нового сотрудника в систему.

## Требования
- **Доступность** Страница должна быть доступна пользователям с соответствующими правами доступа.
- **Интуитивный интерфейс** Интерфейс страницы должен быть простым и понятным, обеспечивая легкость в использовании.
- **Отображение данных**
   - Таблица должна отображать всех действующих пользователей с колонками:
     - Табельный номер.
     - Должность.
     - Инициалы.
- **Фильтрация**
   - Возможность переключения между активными и архивными учетными записями.
- **Поиск**
   - Поле для поиска по инициалам или табельному номеру должно быть расположено под заголовками столбцов.
- **Сортировка**
   - Таблица должна сортироваться по табельному номеру.
- **Кнопки управления**
   - Кнопка "В архив" для отправки выбранной учетной записи в архив.
   - Кнопка "Редактировать" для открытия страницы с заполненной формой выбранного сотрудника.
   - Кнопка "Добавить" для открытия страницы с формой для добавления нового пользователя.
- **Общая информация**
   - Над таблицей должна отображаться информация о количестве активных пользователей с возможностью фильтрации по архивным учетным записям.
- **Динамическое отображение**
   - Блок должен динамически обновляться при выборе другого сотрудника из таблицы.
   - По умолчанию должен открываться первый сотрудник в списке.
- **Форма с данными**
   - Блок должен содержать поля для отображения следующих данных:
     - Должность.
     - Табельный номер.
     - Дата приема на работу.
     - Логин.
     - День рождения.
     - Удаленное подключение.
     - Контактные данные (мобильный телефон, адрес проживания, адрес по прописке, email).
     - Иконка пользователя.
- **Дополнительные блоки**
   - Блок с характеристикой сотрудника.
   - Блок "Примечание" для ввода и отображения заметок о сотруднике.
- **Объем выполненных работ**
   - Таблица должна отображать информацию о выполненных работах выбранного сотрудника с колонками:
     - Дата.
     - № Заказа.
     - Изделие.
     - Операция.
     - Норма ч.
     - Кол-во, ч.
     - Итого, ч.
- **Фильтрация**
   - Возможность фильтрации данных по дате.
- **Общий итог**
   - Отображение общего итога по выполненным работам.

## Описание
Таблица "Сотрудники" отображает всех действующих пользователей системы с возможностью переключения на архивные записи.\
Она предоставляет пользователям удобный интерфейс для управления учетными записями сотрудников, включая фильтрацию, поиск и редактирование данных.

### Структура
- **Табельный номер**
    - уникальный идентификатор сотрудника.
- **Должность**
    - текущая должность сотрудника.
- **Инициалы**
    - ФИО сотрудника.
- **Поиск**
    - Расположен под описанными выше колонками и занимает всю их ширину.
    - Позволяет искать сотрудника по "Инициалам" или "Табельному номеру".
    - Включает возможность просмотра истории запросов поиска.
    - Используется placeholder для подсказки пользователю допустимых значений поиска.

### Особенности
- **Интуитивный интерфейс**
    - Удобное отображение данных в виде таблицы, что позволяет быстро просматривать и управлять информацией о сотрудниках.
- **Фильтрация и поиск**
    - Возможность фильтрации по архивным записям и поиска по инициалам или табельному номеру для быстрого нахождения нужного сотрудника.
- **Динамическое обновление информации**
    - Изменение текста и количества пользователей в зависимости от выбранного фильтра.
- **Общая информация**
    - Над таблицей отображается количество активных пользователей в формате "Активные пользователи: *Количество*". При клике на эту информацию таблица фильтруется по архивным учетным записям, а текст изменяется на "Архив: *Количество*".

### Функционал
- **Отображение списка сотрудников**
    - Таблица показывает всех действующих пользователей с возможностью переключения на архивные записи.
- **Поиск**
    - Пользователь может вводить данные в поле поиска, и таблица обновляется, отображая только тех сотрудников, которые соответствуют введенным критериям.
- **Кнопки управления** (расположены под таблицей):
    - **В архив**: При нажатии на кнопку "В архив" выбранная учетная запись отправляется в архив с уведомлением об успешности операции. При фильтрации таблицы по архиным учетным записям кнопка меняет названия на "В Активные" и при нажатии отправляет выбранную архивную учетную запись в список активных с уведомлением об успешности операции.
    - **Редактирование**: Кнопка "Редактировать" открывает страницу с заполненной формой для изменения данных выбранного сотрудника.
    - **Добавление**: Кнопка "Добавить" открывает страницу с формой для ввода данных нового пользователя.

## Описание
Блок с подробной информацией о сотруднике динамически отображает данные выбранного сотрудника из таблицы "Сотрудники".\
Он включает заголовок с инициалами сотрудника и датой последнего посещения сайта.\
По умолчанию открывается информация о первом сотруднике в списке.\
Блок выполнен в виде формы, содержащей ключевые данные о сотруднике, а также дополнительные блоки с характеристиками и примечаниями.

### Особенности
- **Динамическое обновление**
    - Информация в блоке обновляется автоматически при выборе другого сотрудника из таблицы "Сотрудники".
- **Интуитивный интерфейс**
    - Блок выполнен в виде формы, что облегчает восприятие данных.
- **Полный набор данных**
    - Отображает все ключевые данные о сотруднике, включая контактную информацию и характеристики.
- **Дополнительные блоки**
    - Включает блоки с характеристикой сотрудника и примечаниями для более детального описания.

### Отображение информации о сотруднике
- **Заголовок**
    - Заголовок с инициалами выбранного сотрудника и датой последнего посещения сайта.
- Поля формы для отображения следующих данных:
    - **Должность**.
    - **Табельный номер**.
    - **Дата приема на работу**.
    - **Логин**.
    - **День рождения**.
    - **Удаленное подключение**: чекбокс с отметкой возможности данного пользователя подключаться удаленно.
    - **Контактные данные**:
      - **Мобильный телефон**.
      - **Постоянный адрес проживания**.
      - **Адрес по прописке**.
      - **Email**.
    - Иконка пользователя.

## Описание
Таблица "Объем выполненных работ" расположена под информацией о сотруднике и отображает список выполненных работ выбранного сотрудника из таблицы "Сотрудники".\
Таблица динамически обновляется при выборе другого сотрудника и предоставляет возможность фильтрации данных по дате или периоду.

### Структура
- **Дата**
    - Отображает дату выполнения работы.
- **№ Заказа**
    - Отображает номер заказа, для которого выполнялась работа.
- **Изделие**
    - Наименование изделия, связанного с выполненной работой.
- **Операция**
    - Тип операции, выполненной для изделия.
- **Норма ч.**
    - Количество часов по норме для выполнения данной операции.
- **Кол-во, ч.**
    - Фактическое количество затраченных часов на выполнение работы.
- **Итого, ч.**
    - Общее количество часов, затраченных на выполнение работы.

### Особенности
- **Динамическое обновление**
    - Таблица автоматически обновляется при выборе другого сотрудника, отображая соответствующую информацию о выполненных работах.
- **Фильтрация по дате** (Подключен в виде компонента "Фильт по датам REQ003")
    - Возможность фильтрации данных по дате или периоду для более точного анализа выполненных работ.

### Функционал
- **Отображение списка выполненных работ**
    - Таблица показывает все работы, выполненные выбранным сотрудником, с возможностью быстрого доступа к информации.
- **Фильтрация**
    - Пользователь может фильтровать данные по дате или периоду, чтобы увидеть только те работы, которые соответствуют заданным критериям.
- **Итоговая информация**
    - Внизу таблицы отображается общий итог по всем выполненным работам, что позволяет пользователю быстро оценить общую нагрузку на сотрудника.
