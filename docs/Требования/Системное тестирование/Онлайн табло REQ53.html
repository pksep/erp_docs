Онлайн табло REQ53

Путь​

Производство (/production) -> Онлайн табло (/online-board)

Описание​

Страница "Онлайн табло" предназначена для отображения состояния деталей и сборок на различных этапах операций.
Она предоставляет пользователям актуальную информацию о текущем статусе выполнения заказов, что позволяет эффективно управлять производственными процессами и отслеживать прогресс.

Особенности​

  • Интуитивный интерфейс
    • Страница имеет простой и понятный интерфейс, что облегчает восприятие информации о состоянии деталей и сборок.
  • Многоуровневая фильтрация
    • Пользователи могут использовать несколько фильтров для поиска информации, что позволяет быстро находить нужные данные.

Функционал​

  • Отображение состояния деталей и сборок
    • Страница показывает текущее состояние деталей и сборок на различных этапах операций.
  • Фильтрация данных
    • По заказам склада: Позволяет пользователям искать информацию о деталях и сборках, связанных с заказами склада.
    • По заказам покупателей: Позволяет фильтровать данные по заказам покупателей.
    • По сотруднику: Пользователи могут фильтровать данные по сотруднику, ответственному за выполнение операции.
    • По операции: Возможность фильтрации по конкретной операции.
    • По родительской сущности: Позволяет отфильтровать таблицу по родительской сущности.
    • По наименованию или артикулу: Быстрый поиск по наименованию или артикулу детали или сборки.
  • Печать содержания таблицы
    • Кнопка для отправки содержания таблицы на предпросмотр печати, позволяя пользователю распечатать информацию о состоянии деталей и сборок.

Требования​

  • Заголовок страницы
    • Заголовок должен отображать текст "Онлайн табло".
  • Таблица "Онлайн табло" - Должна содержать следующие столбцы:
    • №
    • Изображение
    • Обозначение
    • Наименование
    • Дата план. отгрузки
    • Заказано
    • Операции
    • Время выполнения
  • Фильтры
    • Должны быть доступны фильтры для сортировки по заказам, операциям и сотрудникам, включая свитчеры и модальные окна для выбора.
  • Отображение данных
    • Таблица должна отображать актуальную информацию о состоянии деталей и сборок, включая даты отгрузки и количество заказанных единиц.
  • Фильтрация данных
    • Пользователь должен иметь возможность фильтровать данные по:
    • Заказам (по плану, по заказу покупателей, по производственному заданию).
    • Сотрудникам (выбор сотрудника из модального окна).
    • Операциям (выбор типа операции из модального окна).
    • Родителям (выбор родителя из модального окна).
  • Поиск
    • Должно быть поле ввода для быстрого поиска по артикулу или наименованию.
  • Печать содержания
    • Кнопка "Печать" должна отправлять содержание таблицы на предпросмотр печати, позволяя пользователю распечатать информацию.

Описание​

Таблица "Онлайн табло" предназначена для отображения списка деталей или сборок с краткой информацией о них.
Она отображает краткую информацию о сущности, включает самую раннюю дату плановой отгрузки, количество заказанных сущностей, а также состояние каждой сущности на каждом этапе технологической операции.
Это позволяет пользователям быстро оценить текущее состояние выполнения заказов и производственных процессов.

Структура таблицы​

  • №
    • Порядковый номер в таблице.
  • Изображение
    • Изображение сущности, позволяющее визуально идентифицировать детали или сборку.
  • Обозначение
    • Уникальное обозначение сущности.
  • Наименование
    • Полное название сущности.
  • Дата план. отгрузки
    • Самая ранняя дата плановой отгрузки из заказов покупателей; если их нет, то из заказов склада.
  • Заказано
    • Количество заказанных сущностей по всем заказам склада.
  • Операции
    • Отображает все технологические операции, которые должна пройти сущность для её изготовления, а также количество проставленных отметок о выполнении на каждой операции из общих заказов склада.
  • Время выполнения
    • Общее время выполнения для всех заказов склада каждой сущности.

Особенности​

  • Вкладки "Металлообработка" и "Сборка"
    • Над таблицей расположены вкладки, позволяющие переключать содержание таблицы между деталями и сборками, что помогает избежать путаницы.
    • По умолчанию выбранна вкладка "Металлообратка".
  • Поле ввода поиска
    • Строка поля ввода для быстрого поиска по артикулу или наименованию, что упрощает нахождение нужной информации.
    • Расположена над таблицей "Онлайн табло".
  • Интуитивный интерфейс
    • Удобное представление данных в виде таблицы, что облегчает работу с состоянием деталей и сборок.
  • Сортировка таблицы
    • По умолчанию таблица сортируется по наиболее ранней дате плановой отгрузки из заказов покупателей; при отсутствии таких заказов - из заказов склада.

Функционал​

  • Отображение информации о деталях и сборках
    • Таблица показывает актуальную информацию о состоянии каждой сущности, включая даты отгрузки и количество заказанных единиц.
  • Фильтрация и поиск
    • Пользователи могут использовать вкладки и поле ввода для фильтрации данных и быстрого поиска нужной информации.
  • Печать содержания
    • Кнопка "Печать", расположенная внизу экрана с фиксированным положением, позволяет пользователям распечатать содержание таблицы для дальнейшего использования.

Описание​

Фильтр по заказам на странице "Онлайн табло" выполнен в виде свитчера и позволяет пользователям фильтровать таблицу по выбранному заказу.
Это обеспечивает гибкость в отображении информации и помогает быстро находить нужные данные о деталях и сборках.

Особенности​

  • Интуитивный интерфейс
    • Свитчер позволяет легко переключаться между различными режимами фильтрации, что делает процесс поиска более удобным.
  • Установка по умолчанию
    • Режим "По плану" установлен по умолчанию, что обеспечивает автоматическую сортировку по наиболее ранней дате плановой отгрузки из заказов покупателей, а при отсутствии таких заказов — из заказов склада.
  • Модальные окна для выбора заказов
    • При выборе режимов "По заказу покупателей" и "По производственному заданию" открываются модальные окна со списками соответствующих заказов, что упрощает процесс выбора.

Функционал​

  • Фильтрация по плану
    • При выборе режима "По плану" таблица "Онлайн табло" автоматически сортируется по наиболее ранней дате плановой отгрузки, что позволяет пользователям видеть заказы в упорядоченном виде.
  • Фильтрация по заказу покупателей
    • При нажатии на этот режим открывается модальное окно со списком всех заказов покупателей. После выбора заказа таблица фильтруется по заказанным сущностям в этом заказе, что позволяет пользователям быстро находить нужные детали или сборки.
  • Фильтрация по производственному заданию
    • При выборе этого режима открывается модальное окно со списком заказов склада. После выбора производственного задания таблица "Онлайн табло" фильтруется по выбранному заданию, что помогает пользователям отслеживать выполнение конкретных производственных задач.

Описание​

Фильтр "Сортировка по операциям или сотрудникам" позволяет пользователям отфильтровать таблицу "Онлайн табло" по выбранному сотруднику или по этапу операции.
Этот фильтр выполнен в виде свитчера, что обеспечивает удобство в использовании и быструю навигацию по данным.

Особенности​

  • Интуитивный интерфейс
    • Свитчер позволяет легко переключаться между различными режимами фильтрации, что делает процесс поиска более удобным и интуитивно понятным.
  • Установка по умолчанию
    • Режим "Все" установлен по умолчанию, что сбрасывает данный фильтр и отображает все сущности в таблице.
  • Модальные окна для выбора
    • При выборе режимов "По сотруднику" и "По операции" открываются соответствующие модальные окна, что упрощает процесс выбора.

Функционал​

  • Сброс фильтра
    • При выборе режима "Все" таблица сбрасывает фильтрацию и отображает все сущности, что позволяет пользователям видеть полную информацию.
  • Фильтрация по сотруднику
    • При нажатии на режим "По сотруднику" открывается модальное окно "Выбрать сотрудника", где пользователи могут выбрать сотрудника, который участвовал на этапе операции. После выбора, таблица отображает все сущности, где участвовал данный сотрудник, а операция, за которую он ответственен, будет выделена в таблице.
  • Фильтрация по операции
    • При выборе режима "По операции" открывается модальное окно с типами операций. В зависимости от выбранной вкладки:
      • Для вкладки "Металлообработка" открывается модальное окно "Детали по операциям".
      • Для вкладки "Сборка" открывается модальное окно "Сборочные единицы по операциям".
    • Пользователи могут выбрать тип операции, после чего таблица отфильтруется и отобразит сущности, где данная операция присутствует. Выбранная операция будет выделена в таблице, что облегчает визуальное восприятие информации.

Описание функции "Сортировка по родителю"​

Кнопка "Выбрать" связана с функцией сортировки по родительскому объекту и предназначена для выбора и установки конкретного объекта (СБ, Изд, Д) в качестве критерия фильтрации.
При нажатии на эту кнопку открывается модальное окно "База Продукции", которое позволяет пользователю выбрать необходимый объект из доступных вариантов.
Пример изображения МетеллообработкаКнопка

Особенности​

  • Интуитивное обозначение
    • Над кнопкой выводится текст "Выбрать Объект (СБ, Изд, Д)", что четко указывает на ее функциональность и доступные опции.
  • Модальное окно "База Продукции"
    • Окно предоставляет пользователю список или дерево объектов, из которых можно выбрать нужный элемент для сортировки.

Функциональность​

  • Открытие модального окна
    • Нажатие на кнопку "Выбрать" активирует модальное окно "База Продукции", где пользователь может выбрать объект для сортировки.
  • Выбор объекта
    • Пользователь выбирает один из предложенных объектов (СБ, Изд, Д), который затем используется как критерий для сортировки данных.
  • Отобразить выбранные позиции
    • При выборе сущности для фильтрации появляется скрытая кнопка "Отобразить выбранные позиции". При нажатии на неё отображается таблица со списком выбранных позиций, включающая Артикул и Наименование, а кнопка меняет название на "Скрыть выбранные позиции". При клике скрывает таблицу выбранных сущностей. Таблица разделена на категории для удобства обозначения типа сущности.
