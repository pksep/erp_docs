## Путь
Архив (/archive)

---
## Описание
Страница "Архив" предназначена для хранения данных, перенесенных в архив.\
Она предоставляет пользователям удобный доступ к информации о ранее завершенных или отмененных заказах, материалах и других сущностях.\
Страница выполнена в виде таблицы, что позволяет легко просматривать и управлять архивированными данными.

### Особенности
- **Таблица с данными**
    - Архив представлен в виде таблицы, что обеспечивает удобный просмотр и навигацию по архивированным данным.
- **Свитчеры для фильтрации**
    - Страница включает различные вкладки, которые позволяют пользователям фильтровать данные по различным критериям.
- **Интуитивный интерфейс**
    - Простота и ясность интерфейса делают взаимодействие с архивом легким и понятным для пользователей.

### Функционал
- **Просмотр архивированных данных**
    - Пользователи могут просматривать все архивированные записи, что позволяет им быстро находить нужную информацию.
- **Фильтрация данных**
    - Возможность фильтрации данных по различным критериям с помощью вкладок, что упрощает поиск конкретных записей.
- **Поиск по архиву**
    - Пользователи могут использовать функцию поиска для быстрого нахождения определенных записей в архиве.
- **Доступ к деталям**
    - При необходимости пользователи могут открывать подробную информацию о каждой архивированной записи, что помогает в анализе и управлении данными.

## Требования
- **Свитчер категорий** - Должен предоставлять пользователю возможность выбора одной из следующих категорий:
    - Изделия
    - Сборки
    - Детали
    - Материалы
    - Пользователи
    - Инструмент и оснастка
    - Оборудование
    - Техника и инвентарь
    - Поставщик
    - Покупатели
    - Файлы
- **Таблица данных**
   - Должна отображать общее количество сущностей над таблицей.
   - Заголовок таблицы должен соответствовать выбранной категории.
   - Столбцы таблицы должны изменяться в зависимости от выбранной категории, включая:
     - Изделия: "Обозначение", "Артикул", "Наименование"
     - Сборки: "Обозначение", "Наименование"
     - Детали: "Обозначение", "Наименование"
     - Материалы: "Тип", "Наименование"
     - Пользователи: "Табельный номер", "Должность", "Инициалы"
     - Инструмент и оснастка: "Тип", "Наименование"
     - Оборудование: "Наименование"
     - Техника и инвентарь: "Наименование"
     - Поставщик: "ИНН", "Наименование поставщика"
     - Покупатели: "ИНН", "Наименование покупателя"
     - Файлы: "№", "Тип", "Наименование", "Дата", "Примечание"
- **Поиск**
   - Должен быть расположен под заголовками столбцов и занимать всю их ширину.
   - Должен включать возможность просмотра истории запросов поиска.
   - Должен использовать placeholder для подсказки пользователю допустимых значений поиска.
- **Выбор категории**
   - Пользователь должен иметь возможность выбирать категорию из свитчера, что инициирует отображение соответствующей таблицы с архивными записями.
- **Просмотр архивных данных**
   - Пользователь должен иметь возможность просматривать архивные записи в таблице, включая общее количество сущностей.
- **Поиск по записям**
   - Пользователь должен иметь возможность использовать поле поиска для быстрого нахождения конкретных записей по обозначению, наименованию или другим критериям.
- **Доступ к информации**
   - При клике на записи Изделий, Сборки или Деталей пользователь должен получать доступ к модальному окну "Краткая информация", где отображаются краткая информация выбранной сущности.
- **Интерактивность**
   - Пользователь должен иметь возможность легко взаимодействовать с таблицей, выбирая категории и подкатегории.

## Описание
Страница "Архив" предназначена для хранения и управления данными, перенесенными в архив.\
Она обеспечивает пользователям удобный доступ к информации о ранее завершенных или отмененных заказах, материалах, пользователях и других сущностях.\
При открытии страницы пользователю предоставляется выбор категорий в виде свитчера.\
При нажатии на любую из категорий появляется таблица данных с архивными записями выбранной категории, что позволяет удобно просматривать и анализировать информацию.

### Структура Свитчер категорий позволяет пользователю выбирать одну из следующих категорий:
- **Изделия**
    - Архивированные изделия с возможностью просмотра их характеристик.
- **Сборки**
    - Архивированные сборочные единицы, позволяющие отслеживать завершенные проекты.
- **Детали**
    - Архивированные детали, используемые в производственных процессах.
- **Материалы**
    - Архивированные материалы, которые были использованы или заказаны.
- **Пользователи**
    - Архивированные записи о пользователях системы.
- **Инструмент и оснастка**
    - Архивированные инструменты и оснастка, используемые в производстве.
- **Оборудование**
    - Архивированные записи об оборудовании.
- **Техника и инвентарь**
    - Архивированные записи о технике и инвентаре.
- **Поставщик**
    - Архивированные записи о поставщиках.
- **Покупатели**
    - Архивированные записи о покупателях.
- **Файлы**
    - Архивированные файлы, связанные с различными процессами.

### Структура Таблицы данных для каждой категории отображается таблица с архивными записями, содержащая следующие элементы:
- **Общее количество сущностей**
    - Отображается над таблицей, позволяя пользователю видеть количество записей в архиве.
- **Заголовок таблицы**
    - Соответствующий заголовок для каждой категории, например, "Изделия", "Сборочная единица" и т.д.
- **Столбцы таблицы** - В зависимости от категории, таблица может содержать различные столбцы, такие как:
    - **Изделия**: "Обозначение", "Артикул", "Наименование"
    - **Сборки**: "Обозначение", "Наименование"
    - **Детали**: "Обозначение", "Наименование"
    - **Материалы**: "Тип", "Наименование"
    - **Пользователи**: "Табельный номер", "Должность", "Инициалы"
    - **Инструмент и оснастка**: "Тип", "Наименование"
    - **Оборудование**: "Наименование"
    - **Техника и инвентарь**: "Наименование"
    - **Поставщик**: "ИНН", "Наименование поставщика"
    - **Покупатели**: "ИНН", "Наименование покупателя"
    - **Файлы**: "№", "Тип", "Наименование", "Дата", "Примечание"
- **Поиск**
    - Расположен под заголовками столбцов и занимает всю их ширину.
    - Включает возможность просмотра истории запросов поиска.
    - Используется placeholder для подсказки пользователю допустимых значений поиска.

### Особенности
- **Интуитивный интерфейс**
    - Простота и ясность интерфейса делают выбор категорий и просмотр архивных данных легким и понятным для пользователей.
- **Динамическое обновление**
    - При выборе категории таблица автоматически обновляется, отображая соответствующие архивные записи.
- **Модальное окно "Краткая информация"**
    - При клике на сущность Изделия, Сборки или Детали открывается модальное окно с краткой информацией о выбранной сущности, что позволяет пользователям быстро получать нужные данные.
- **Фильтрация**
    - Возможность фильтрации данных в таблицах по категориям, что упрощает поиск нужной информации.

### Функционал
- **Выбор категории**
    - Пользователь может выбрать категорию из свитчера, что инициирует отображение соответствующей таблицы с архивными записями.
- **Просмотр архивных данных**
    - Пользователь может просматривать архивные записи в таблице, включая общее количество сущностей.
- **Поиск по записям**
    - Пользователь может использовать поле поиска для быстрого нахождения конкретных записей по обозначению, наименованию или другим критериям.
- **Доступ к информации**
    - При клике на запись пользователь получает доступ к модальному окну "Краткая информация", где отображаются детали выбранной сущности.
- **Интерактивность**
    - Пользователи могут легко взаимодействовать с таблицей, выбирая категории и подкатегории, что делает процесс заказа более удобным.
