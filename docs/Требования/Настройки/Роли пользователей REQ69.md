## Путь
Настройки (/settings) -> 12. Роли пользователей (/rolesuser)

---
## Описание
Страница "Роли пользователей" предназначена для управления доступами пользователей в системе.\
Она позволяет администраторам настраивать права доступа для различных ролей, обеспечивая гибкость в управлении функционалом системы.\
Страница включает в себя выпадающее меню (dropdown), где можно выбрать одну из внесенных в систему ролей пользователей, а также содержит список различных функций системы, к которым можно изменять доступ.

### Особенности
- **Интуитивный интерфейс**
    - Страница имеет простой и понятный интерфейс, что облегчает пользователям навигацию и управление доступами.
- **Dropdown меню**
    - Пользователи могут легко выбирать роли из выпадающего меню, что упрощает процесс управления доступами.
- **Toggle для доступа**
    - Доступ к функциям системы реализован в виде переключателей (toggle), что позволяет быстро изменять права доступа к чтению и редактированию.

### Функционал
- **Выбор роли**
    - Пользователи могут выбирать роль из выпадающего меню, чтобы просмотреть и изменить доступы для этой роли.
- **Обновление списка доступов**
    - После выбора роли автоматически обновляется список доступов, отображая текущие права для выбранной роли.
- **Список функций системы**
    - На странице отображается список различных функций системы, к которым можно настраивать доступ.
- **Изменение доступа** - Каждая функция имеет два переключателя (toggle):
    - **Чтение**: Позволяет включать или отключать доступ к чтению для выбранной роли.
    - **Изменение**: Позволяет включать или отключать доступ к изменению для выбранной роли.
- **Сохранение изменений**
    - После изменения доступов пользователи могут сохранять настройки, что позволяет обновить права доступа для выбранной роли.
- **Информационные подсказки**
    - Страница может содержать подсказки, объясняющие, какие права доступа могут быть изменены и как это повлияет на пользователей с данной ролью.
- **Сохранение изменений**
    - После изменения доступов пользователи могут сохранять настройки, что позволяет обновить права доступа для выбранной роли.

## Требования
- **Доступность**
    - Компонент "Список доступа" должен быть доступен пользователям с соответствующими правами доступа (например, администраторам).
- **Интуитивный интерфейс**
    - Интерфейс компонента должен быть простым и понятным, обеспечивая легкость в использовании.
- **Список доступа**
    - Должен отображать различные разделы системы с текущим состоянием прав доступа для выбранной роли.
    - Доступы должны быть организованы в блоки по функциональной нагрузке.
- **Toggle для прав доступа** - Каждая функция системы должна иметь два переключателя (toggle):
    - **Чтение**: Для включения или отключения доступа к чтению.
    - **Изменение**: Для включения или отключения доступа к изменению.
- **Кнопка "Обновить"**
    - Должна быть доступна для применения изменений прав доступа.
    - Должна быть четко обозначена и расположена в удобном месте.
- **Динамическое обновление**
    - При выборе роли из выпадающего меню список доступа должен автоматически обновляться, отображая текущие права доступа для выбранной роли.
- **Изменение прав доступа**
    - Пользователь должен иметь возможность изменять права доступа, используя переключатели (toggle) для "Чтения" и "Изменения".
- **Сохранение изменений**
    - При нажатии на кнопку "Обновить" изменения прав доступа должны сохраняться в системе.
- **Выбор роли**
    - Пользователь должен иметь возможность выбирать роль из выпадающего меню, чтобы просмотреть и изменить доступы для этой роли.
- **Информационные подсказки**
    - Компонент должен содержать подсказки, объясняющие, какие права доступа могут быть изменены и как это повлияет на пользователей с данной ролью.
