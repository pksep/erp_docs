Действия пользователей REQ71

Путь​

Настройки (/settings) -> 17. История действий (/actions)

Описание​

Страница "Действия пользователей" предназначена для отображения истории действий пользователей в системе.
Она позволяет пользователям с определенными правами отслеживать действия, выполненные пользователями, что способствует повышению прозрачности и безопасности работы системы.
Страница выполнена в виде таблицы с различными вкладками по категориям для упрощения поиска истории, а также предоставляет возможность фильтрации истории по сотрудникам.

Особенности​

  • Интуитивный интерфейс
    • Страница имеет простой и понятный интерфейс, что облегчает пользователям навигацию и поиск нужной информации.
  • Табличное представление
    • Действия пользователей представлены в виде таблицы, что позволяет легко просматривать и анализировать информацию о действиях.
  • Вкладки для упрощения поиска
    • Наличие различных вкладок позволяет пользователям быстро переключаться между различными категориями действий, упрощая поиск нужной информации.
  • Фильтрация по сотрудникам
    • Возможность фильтрации истории действий по конкретным сотрудникам, что позволяет быстро находить действия, выполненные определенными пользователями.

Функционал​

  • Просмотр истории действий
    • Пользователи могут просматривать список всех действий, выполненных пользователями, включая детали, такие как дата, время, тип действия и ответственный сотрудник.
  • Фильтрация истории
    • Пользователи могут фильтровать историю действий по сотрудникам, что позволяет быстро находить нужные записи.
  • Вкладки для категорий действий
    • Страница может содержать вкладки, которые группируют действия по различным категориям (например, "Добавление", "Редактирование", "Удаление"), что упрощает навигацию.
  • Поиск по действиям
    • Возможность поиска по ключевым словам или параметрам, что позволяет быстро находить конкретные действия в истории.
  • Информационные подсказки
    • Страница может содержать подсказки или описания, которые помогают пользователям понять, какие действия были выполнены и как это повлияло на систему.

Требования​

  • Доступность
    • Страница "Действия пользователей" должна быть доступна пользователям с соответствующими правами доступа (например, администраторам или менеджерам).
  • Интуитивный интерфейс
    • Интерфейс страницы должен быть простым и понятным, обеспечивая легкость в использовании.
  • Таблица действий пользователей - Должна отображать следующие столбцы:
    • Дата и Время: Дата и время совершения действия.
    • Инициатор: ФИО сотрудника, совершившего действие.
    • Тип объекта: Тип категории, к которой относится совершенное действие.
    • Действие: Описание совершенного действия.
    • Подробно: Иконка, при наведении на которую отображается tooltip с подробной информацией о совершенном действии.
  • Свитчеры для фильтрации - Должны быть расположены слева от таблицы и позволять переключаться между различными категориями действий:

    • Должна быть доступна для отмены всех примененных фильтров к таблице.
  • Просмотр истории действий
    • Пользователь должен иметь возможность просматривать список всех действий, совершенных пользователями, включая детали, такие как дата, инициатор, тип объекта и описание действия.
  • Фильтрация истории
    • Пользователь должен иметь возможность фильтровать историю действий по сотрудникам, используя иконку фильтрации, которая открывает модальное окно для выбора сотрудника.
  • Фильтрация по типу объекта

  • Поиск по действиям

Описание фильтрации по "Сотрудник"​

Функция фильтрации по сотруднику представляет собой интерактивную иконку, расположенную в интерфейсе таблицы данных.
Эта иконка служит точкой входа для быстрой и наглядной фильтрации данных по конкретным сотрудникам.

Особенности​

  • Интуитивный интерфейс
    • Иконка фильтрации четко обозначена и легко доступна, что делает её использование простым и понятным для пользователей.
  • Модальное окно
    • При активации иконки открывается модальное окно "Выбрать сотрудника", которое предоставляет пользователю список доступных сотрудников для выбора.
  • Выбор сотрудников
    • В модальном окне пользователь может выбрать сотрудника, используя поиск по различным параметрам, таким как табельному номеру или инициалам. После выбора сотрудника таблица сборки (Table 1) будет отфильтрована, отображая только те действия, которые связаны с выбранными сотрудником.

Функциональность​

  • Применение фильтра
    • После выбора сотрудников, таблица автоматически обновляется, отображая только те записи, которые связаны с выбранными сотрудниками.
  • Отмена фильтрации
    • Пользователь имеет возможность отменить выбор или закрыть модальное окно без применения фильтрации.

Описание​

Таблица "Действия пользователей" предназначена для отображения истории действий, совершенных пользователями в системе.
Она позволяет пользователям с определенными правами отслеживать действия, выполненные пользователями, и анализировать их.
