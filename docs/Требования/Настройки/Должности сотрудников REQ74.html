Должности сотрудников REQ74

Путь​

Настройки (/settings) -> 13. Должности сотрудников (/dolznuser)

Требования​

  • Доступность
    • Страница "Должности сотрудников" должна быть доступна пользователям с соответствующими правами доступа (например, администраторам или менеджерам).
  • Интуитивный интерфейс
    • Интерфейс страницы должен быть простым и понятным, обеспечивая легкость в использовании.
  • Таблица должностей - Должна отображать следующие столбцы:
    • №: Порядковый номер в таблице.
    • Название должности: Отображает название должности.
  • Компоненты управления - Должны быть доступны следующие кнопки:
    • Создать: Для добавления новой должности.
    • Редактировать: Должна быть скрыта по умолчанию и появляться только при выборе в таблице какой-либо должности.
    • В архив: Должна быть скрыта по умолчанию и появляться только при выборе в таблице какой-либо записи.
  • Добавление новой должности
    • При нажатии на кнопку "Создать" должно открываться модальное окно с формой для заполнения.
    • Форма должна содержать поля:
    • Наименование: Поле ввода для названия должности.
    • Описание роли: Поле textarea для ввода описания роли.
    • Должны быть доступны кнопки "Сохранить" и "Отменить".
  • Редактирование существующей должности
    • При нажатии на кнопку "Редактировать" должно открываться модальное окно с заполненной формой выбранной должности.
  • Архивирование должности
    • При нажатии на кнопку "В архив" выбранная должность должна отправляться в архив.
  • Увеломения
    • При успешном выполнении операции, должно появляться уведомление об успешности операции.
  • Валидация ввода
    • Поля ввода должны проверять корректность введенных данных (например, на наличие пустых значений).
  • Закрытие модального окна
    • Пользователь должен иметь возможность закрыть модальное окно без сохранения изменений.

Описание​

Страница "Должности сотрудников" предназначена для управления должностями, внесенными в базу данных.
Она позволяет пользователям добавлять новые должности, редактировать существующие или отправлять их в архив.
Страница выполнена в виде таблицы, что обеспечивает удобный доступ к информации о должностях.

Структура таблицы​

  • №
    • Порядковый номер в таблице.
  • Название должности
    • Отображает название должности.

Компоненты управления​

  • Кнопка "Создать"
    • Позволяет добавить новую должность в таблицу.
    • При клике открывается модальное окно с формой для заполнения.
  • Кнопка "Редактировать"
    • По умолчанию скрыта и появляется только при выборе в таблице какой-либо должности.
    • При клике открывается модальное окно с заполненной формой выбранной должности для редактирования.
  • Кнопка "В архив"
    • По умолчанию скрыта и появляется только при выборе в таблице какой-либо записи.
    • При клике отправляет выбранную должность из таблицы в архив и отображает уведомление об успешной операции.

Особенности​

  • Интуитивный интерфейс
    • Страница имеет простой и понятный интерфейс, что облегчает пользователям навигацию и управление должностями.
  • Табличное представление
    • Должности представлены в виде таблицы, что позволяет легко просматривать и анализировать информацию.
  • Динамическое управление
    • Кнопки "Редактировать" и "В архив" появляются только при выборе соответствующей записи, что предотвращает случайные действия.

Функционал​

  • Просмотр должностей
    • Пользователи могут просматривать список всех должностей, внесенных в базу данных.
  • Добавление новой должности
    • Пользователи могут добавлять новые должности через модальное окно, заполняя необходимые поля.
  • Редактирование существующей должности
    • Пользователи могут редактировать информацию о выбранной должности через модальное окно.
  • Архивирование должности
    • Пользователи могут отправлять выбранные должности в архив.
  • Уведомление
    • При успешных операциях всплывает уведомление об успешности операции.

Форма создания\редактирования должности​

Форма представлена в виде модального окна с выделением основных полей для заполнения. Она открывается по нажатию кнопки "Создать", после чего появляется пустая форма с заголовком "Добавление должности". При нажатии на кнопку "Редактировать" открывается форма "Редактировать должность", в которой уже заполнены данные, введенные при создании выбранной должности.

Структура формы​

  • Наименование
    • Поле ввода для названия должности с подсказкой Placeholder "Пример: Слесарь".
    • Ограничение не более 255 символов.
  • Описание роли
    • Поле textarea для ввода описания роли.
    • Ограничение не более 255 символов.
  • Кнопки:
    • Сохранить: Сохраняет введенные данные и добавляет новую должность в таблицу.
    • Отменить: Закрывает модальное окно без сохранения изменений.
