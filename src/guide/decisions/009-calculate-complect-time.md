# Техническое описание: Логика расчета начала работ "Комплектация сборок и изделий"

## 1. Обзор

Данный алгоритм описывает процесс вычисления ключевых временных меток (`start_time`, планируемая дата готовности, `plan_ready_time`) для первой производственной операции технологического процесса — **Комплектации**.

Поскольку комплектация физически не может начаться до того, как будут готовы все составные части, логика вычислений базируется на анализе состояния дочерних элементов (деталей и подсборок первого уровня) и синхронизируется с графиком последующих сборочных операций.

---

## 2. Общие принципы расчета

- **Зависимость от дочерних элементов:** Расчет стартовой даты операции всегда отталкивается от самой поздней даты готовности необходимого компонента (детали или сборочной единицы). Сборка начинается только тогда, когда доступен полный комплект.
- **Фокус на финальные операции ПЗ:** Готовность ребенка определяется по его последней технологической операции (операция ТП с максимальным `idx` в техпроцессе).
- **Производственный календарь:** Все добавления времени (например, времени на комплектацию) рассчитываются через утилиту `calculateEndDate`, которая строго учитывает рабочие смены и производственный график.

---

## 3. Детальное описание этапов расчета

### Этап 1: Расчет "Начала работ" (`start_time`)

Определяет фактическое время, когда комплектовщик сможет приступить к сбору деталей по конкретному Производственному заданию (ПЗ). Назначается на операцию с `idx === 1`.

**Алгоритм:**

1. **Сбор детей:** Система запрашивает всех дочерних элементов первого уровня (Сборки и Детали) для текущей родительской сущности.
2. **Оценка готовности ПЗ детей:** - Для каждого дочернего элемента анализируются все его ПЗ.

- Рассматривается только последняя операция техпроцесса (максимальный `idx`).
- Берется самая поздняя дата ( `calculate_needs_time`) среди этих финальных операций.

3. **Определение старта:** Из полученного списка дат готовности всех детей выбирается **самая поздняя (максимальная) дата**. Именно она становится `start_time` для операции комплектации родительского ПЗ.

**Бизнес-смысл:** Ответ на вопрос «Когда на склад поступят абсолютно все детали, необходимые для начала комплектации этого узла?».

---

### Этап 2: Расчет "Расчетной даты готовности" ( `calculate_needs_time`)

Определяет дату и время расчетного завершения операции.

**Алгоритм:**

1. **Базовая точка:** Берется вычисленное на предыдущем этапе "Начало работ" (`start_time`).
2. **Вычисление трудоемкости:** Нормативное время выполнения операции комплектации умножается на заказанное количество изделий по конкретной ПЗ (`quantity_max`).
3. **Итоговый расчет:** К `start_time` прибавляется полученная трудоемкость с учетом производственного календаря.

**Формула:** `start_time + (время_операции_комплектации * quantity_max)`

---

## 3. Триггеры пересчета

Даты операции "Комплектация" подлежат автоматическому каскадному пересчету при следующих событиях:

1. Изменение расчетной даты готовности (`calculate_needs_time`) у любого из дочерних элементов первого уровня.
2. Изменение количества изделий (`quantity_max`) в текущем ПЗ.
3. Любое изменение времени начала работ (`start_time`) у дочерних компонентов.
4. Изменение спецификации родительской сущности.
