# Расчёт времени начала работ (start_time) для производственных операций

## Контекст

Расчёт `start_time` актуален для двух типов ресурсов:

- **Пользователи** — сборочные операции (назначаются на монтажников/сборщиков)
- **Оборудование** — операции металлообработки (назначаются на станки)

## Точки отсчёта (Base Start Times)

| Ресурс       | Таблица          | Назначение                         |
| ------------ | ---------------- | ---------------------------------- |
| Пользователь | `StartTime`      | Базовое время для сборки           |
| Оборудование | `StartDetalTime` | Базовое время для металлообработки |

При отсутствии записи создаётся новая с `start_time = new Date()`.

## Методы расчёта

| Метод                       | Ресурс       | Операции по |
| --------------------------- | ------------ | ----------- |
| `recalculateStartTime`      | Пользователь | `idx`       |
| `recalculateStartTimeDetal` | Оборудование | `detal_idx` |

### Алгоритм

1. **Упорядочивание:** Операции сортируются по порядку выполнения.
2. **Стартовая точка:**
   - Переданное время (ручное изменение)
   - Минимальное `start_time` среди активных операций
   - Базовое время из `StartTime`/`StartDetalTime`
   - `new Date()` — если ничего нет
3. **Коррекция на календарь:** `calculateEndDate(currentStartTime, 0)` переносит время на рабочие часы.
4. **Итерация по операциям:**
   - **Сборка:** Первая операция получает `currentStartTime`, последующие — от предыдущей с учётом трудоёмкости.
   - **Металлообработка:** Строгая цепочка — каждая следующая операция начинается после завершения предыдущей.

## Бизнес-правила

1. Расчёт **персонализирован** по ресурсу (пользователь/оборудование).
2. Пересечение с **производственным календарём** — задачи не попадают на выходные и нерабочие часы.
3. Ручное изменение порядка или времени **пересчитывает весь последующий хвост** операций для ресурса.

## Триггеры пересчёта

- `setStartTimeUser` / `setStartTimeDetal` — ручное изменение базового времени
- `changeOperationPosDetalIndex` — изменение порядка операций (drag-and-drop)
- Создание/изменение ПЗ, норм времени, количества деталей
