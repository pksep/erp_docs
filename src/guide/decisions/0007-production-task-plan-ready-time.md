# Расчёт плановой даты готовности (plan_ready_time) операций ПЗ

## Контекст

Для обеспечения прозрачности производственного цикла и соблюдения сроков отгрузки, каждой операции в составе производственной задачи (ПЗ) должна быть присвоена плановая дата готовности (`plan_ready_time`). Эта дата указывает крайний срок завершения операции, чтобы не сорвать последующие этапы производства или дату отгрузки изделия заказчику.

Метод `actualReadyPlanDateByEntity` реализует логику «обратного планирования», где срок текущей операции зависит от времени начала следующего звена в цепочке.

## Основные компоненты

### 1. Метод `actualReadyPlanDateByEntity`

Основной управляющий метод, который инициирует пересчёт дат для конкретной сущности (Изделие, Сборочная единица, Деталь).

**Алгоритм работы:**

1. **Поиск операций:** Через `getProductionOperationPosIdsByEntity` находятся все активные позиции операций, связанные с данной сущностью.
2. **Определение «Опорной даты» (Relative Date):** Через `getRelativeDateForEntity` получается минимальная дата из всех доступных источников.
3. **Расчёт внутри цепочки техпроцесса:**
   - Для каждой операции в рамках одной позиции ПЗ:
     - Если есть **следующую** операцию (по `idx`): Плановая дата готовности текущей = `start_time` следующей операции.
     - Если это **последняя** операция: Плановая дата готовности = «Опорная дата».

### 2. Метод `getRelativeDateForEntity`

Определяет опорную дату для расчёта `plan_ready_time`. Возвращает самую раннюю дату из всех доступных источников.

**Источники дат (сравниваются, выбирается минимальная):**

| Источник                   | Типы сущностей | Описание                                                                                               |
| -------------------------- | -------------- | ------------------------------------------------------------------------------------------------------ |
| `getActualTaskRelative`    | detal, cbed    | Минимальное время начала (`start_time`) первых операций в родительских сборках, куда входит сущность   |
| `warehouse_readiness_date` | Все            | Дата готовности на складе (из заказа по складу)                                                        |
| `due_data` ПЗ              | Все            | Минимальная плановая дата выполнения среди неархивных производственных заданий, привязанных к сущности |

**Приоритет:** Из всех доступных источников выбирается **самая ранняя дата**.

**SQL-запрос для due_data ПЗ:**

```sql
SELECT pt.due_data, pt.id AS task_id
FROM production_operation_positions pop
JOIN production_task pt ON pop.production_task_id = pt.id
WHERE pop.id IN (...) AND pt.ban = false AND pop.ban = false
ORDER BY pt.due_data ASC LIMIT 1
```

### 3. Вспомогательные методы

- **`getActualTaskRelative`:** Использует Neo4j для поиска родительских сущностей, возвращает минимальную дату начала среди всех найденных родительских задач.
- **`getProductionOperationPosIdsByEntity`:** Возвращает список ID активных позиций операций для сущности.

## Бизнес-правила

1. **Приоритет сроков:** Срок готовности диктуется самым ранним из доступных ориентиров (родительская сборка, склад, ПЗ).
2. **Цепочка зависимостей:** Любое изменение `start_time` в одной операции вызывает каскадный пересчёт `plan_ready_time` у зависимых операций.
3. **Обработка пустого планирования:** Если ни один источник не предоставляет дату, `plan_ready_time` остаётся `null`.

## Триггеры пересчёта

1. **Создание ПЗ:** Автоматический расчёт для созданной сущности и всех её дочерних элементов.
2. **Архивация (Ban):** При архивации ПЗ или отдельных операций.
3. **Изменение спецификации:** При добавлении или удалении компонентов.
4. **Редактирование графика:** При изменении `start_time` или `due_data` внутри ПЗ.

## Архитектурные решения

1. **Консистентность данных:** SQL-запросы с `DISTINCT ON` гарантируют получение актуальных ID без дублирования.
2. **Интеграция с Neo4j:** Графовая БД используется для поиска связей «родитель-ребёнок» в спецификациях.
3. **Единая точка отсчёта:** Метод `getRelativeDateForEntity` централизует логику получения опорной даты для всех типов сущностей.
