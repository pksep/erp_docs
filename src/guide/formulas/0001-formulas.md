# Понятия и формулы системы управления запасами и производством

## Контекст

Данный документ определяет ключевые понятия и формулы, используемые в модулях управления запасами, производственными заданиями (ПЗ) и прочими заказами в ERP-системе. Четкое понимание этих определений и алгоритмов расчета необходимо для корректного формирования отчетов, планирования закупок и производства, а также для обеспечения прозрачности логики работы системы.

---

## Определения (Понятия)

### <a id="potrebnost-roditelya"></a>Потребность родителя (Parent Requirement)

- **Описание:** Количество сущности (детали, сборки, материала), указанное в спецификации для производства одной единицы родительской сущности (сборки или изделия). Для конечного изделия данный показатель не существует, так как у него нет родителя в производственной цепочке.
- **Применение:** Базовый коэффициент для расчета потребностей в компонентах.

### <a id="gotovnost-k-komplektatsii"></a>Готовность к комплектации (Kitting Readiness)

- **Описание:** Статус, который проверяет, достаточно ли на складе всех необходимых дочерних компонентов для комплектации родительской единицы (сборки или изделия). Статус присваивается, если количество каждого дочернего компонента на складе соответствует или превышает значение, рассчитанное как `"Потребность родителя" * "Требуемое количество родителя"`.
- **Применение:** Определяет возможность запуска процесса сборки/комплектации.

- **Общее:** Необходимая дата производства сущности для отгрузки на склад.

#### Дата планируемой готовности на операцию {#warehouse-readines-date-by-operation}
- **Общее:** Рассчетная дата выполнения на операцию, рассчитывается по формуле:
Начало работ по операции + (подготовительное время + (основное время + вспомогательное) * Кол-во заказанное по ПЗ).

- **Для сборок:** Заказанное кол-во на склад у производства.
- **Для деталей:** Заказанное кол-во на склад у производства.
- **Для материалов:** Заказанное кол-во на склад у поставщика.

#### Заказано на производство {#ordered-to-production}
- **Общее:** "Кол-во сущностей на заказ" - "Оприходованное по заказу кол-во"

#### Единица измерения {#mesuarement}
- **Для (тип сущности):** Единица измерения (тип сущности) 

- **Общее:** Единица измерения сущности.

#### Начало работ {#start-time}
- **Для Изделий:** Начало работ это самая ранняя начало работ на самое раннее производственное задание, на второй операции этой сущности.
- **Для сборок:** Начало работ это самая ранняя начало работ на самое раннее производственное задание, на второй операции этой сущности.
- **Общее:** Начало работ это самая ранняя начало работ на самое раннее производственное задание, на самой первой операции этой сущности.

#### Начало работ по операции {#start-time-by-operation}
- **По оборудованию**:  Очередность выполнения на ПЗ по Оборудованию.
- **По исполнителю**:  Очередность выполнения на ПЗ по Сотруднику.

Имеет несколько вариантов рассчета:
- Если сущность первая в списке выставляется дата установленная пользователем, иначе дата берется из колонки "Дата планируемой готовности на операцию" сущности которая расположена выше.
- По "Дата планируемой готовности на операцию" этой же сущности из пред операции
- По "Начало работ по операции" этой же сущности из след операции

При несоблюдении этих условий будет отображаться знак: "-"

#### Требуемая дата готовности  {#calculated-create-time}
- **Общее:** Требуемая дата на самое последнее производственное задание, дата требуемой готовности на операцию, самой последней операции.

#### Дата требуемой готовности на операцию {#calculated-create-time-by-operation}
- **Общее:** Дата, к которой сущность по операции должна быть готова.

Устанавливается исходя из даты начала работ следующей операции.
Если это последняя операция, то выбирается из даты начала работ второй операции родителя или самой ранней даты плановой готовности склада


#### Расчётная дата изготовления  {#calculate-needs-time}
- **Общее:** Расчетная дата берется из самого крайнего ПЗ этой сущности, самой последней операции из поля дата плановой готовности на операцию.

#### Дельта по времени
- **Определение:** Разница между Дата планируемой готовности на операцию и Дата требуемой готовности на операцию. Показывает загруженность на операцию.

#### Время изг. дефицитных Д/СБ
- **Определение:** Время изготовления дефицитных Деталей и Сборок по спецификации в часах и минутах. Стандартная формула времени для всех операций ТП сущности (подготовительное время + (основное время + вспомогательное) * Кол-во отметок ).

#### Задолженность
- **Определение:** Отображает наличие сущностей у которых начало работ по операции позже текущего дня.

#### Операции
- **Предыдущая операция:** Информация о предыдущей операции технологического процесса.
- **Текущая операция:** Информация о текущей операции технологического процесса.
- **Следующая операция:** Информация о следующей операции технологического процесса.

#### Легенда ячейки операций
![alt text](image.png)
- **1:** Время на выполнение операции.
- **2:** Сделано N из N.
- **3:** Такая иконка появляется, если дата планируемой готовности на операцию позднее даты начала работ на следующую операцию.
- **4:** Иконка материала - появляется только на первой операции только для МО. Показывает есть ли материал на складе для производства или его не хватает.
- **5:** Такая иконка появляется, если в операции отсутствует основное время.
- **6:** Название операции технологического процесса.
- **7:** Дата планируемой готовности на операцию.

#### Цвет фона ячейки
- **Зеленый:** Все отметки о выполнении операции проставлены.
- **Голубой:** Работа начата, но не все отметки о выполнении операции проставлены.
- **Желтый:** Работа по операции не начата. По ней нет отметок о выполнении.

## Формулы расчета

### 1. Общее время {#f1-obshee-vremya}

#### Общее время на сборку {#obshee-vremya-na-sborku}

Сумма всех времен производства дочерних сборок, полученных рекурсивно, с учетом их потребности на родителя.

#### Общее время на МО {#obshee-vremya-na-mo}

Сумма всех времен производства дочерних деталей, полученных рекурсивно, с учетом их потребности на родителя.

### 2. Расчет потребностей {#f2-raschet-potrebnostey}

#### Потребность по Заказам Склада (ЗС) {#potrebnost-po-zakazam-sklada}

`Сумма[("Потребность родителя" * "Потребность по ЗС родителя")]`
_Актуально для деталей, сборок, материалов._

#### Потребность по Заказам Поставщика (для материалов) {#potrebnost-po-zakazam-postavshika}

`Сумма[("Потребность по заказам поставщика")]`

### 3. Расчет остатков {#f3-raschet-ostatkov}

#### Минимальный остаток (для сб/д) {#minimalnyi-ostatok}

`Сумма[("Потребность родителя" * "Минимальный остаток родителя")]`

#### Рекомендуемый остаток {#rekomenduemyi-ostatok}

`"Минимальный остаток" * 3`

#### Остаток с учетом прихода {#ostatok-s-uchetom-prihoda}

- _Для сборок и деталей:_ `Сумма["Кол-во родителя на складе"] - "Минимальный остаток родителя"`
- _Для изделий:_ `Сумма["Минимальный остаток родителя"] - "Кол-во"`

#### Реальный остаток с учетом планируемых отгрузок и производства {#realnyi-ostatok}

- _Для деталей и сборок:_ `"Заказано на производстве" + "Кол-во" - "Потребность по ЗС"`
- _Для изделий:_ `"Заказано на производстве" + "Кол-во" - "Потребность по ЗО"`
- _Для материалов:_ `"Заказано на производстве" + "Кол-во" - "Потребность по заказам поставщика"`

### СВОЕ {#deficit-CBOE}
- **Общее:** По умолчанию `"Рекомендуемое количество"`

### 4. Расчет дефицита {#f4-raschet-defitsita}

#### Дефицит (не может быть < 0) {#defitsit}

`"Кол-во" - "Минимальный остаток" + "Потребность по ЗО"`

#### Дефицит с учетом минимального остатка {#defitsit-s-uchetom-minimalnogo-ostatka}

`"Кол-во" - ("Минимальный остаток" + "Дефицит по ЗС")`

#### Дефицит по ЗО (Задачам на Отгрузки) {#defitsit-po-zo}

- _Для сб/д/материала:_ `Сумма[("Потребность родителя" * "Потребность по ЗО родителя")]`
- _Для изделий:_ `Сумма["Заказанное кол-во по ЗО"]`

#### Дефицит по ПЗ (Производственному Заданию) {#defitsit-po-pz}

`"Кол-во" - "Кол-во в наборах" - ("Осталось сделать по ПЗ" * "Потребность родителя")`

### 5. Расчет производственных показателей {#f5-raschet-proizvodstvennyh-pokazateley}

#### В наборах {#v-naborah}

`Сумма["Потребность родителя" * "Скомплектованное кол-во родителя"]`

#### С учетом прихода (Сб и Изд) {#s-uchetom-prihoda}

`"Остаток с учетом прихода" - "Минимальный остаток"`

#### Заказано на производстве (не может быть < 0) {#zakazano-na-proizvodstve}

`Сумма["Кол-во сущностей на заказ" - "Оприходованное по заказу кол-во"]`

#### Осталось сделать по ПЗ (не может быть < 0) {#ostalos-sdelat-po-pz}

`"Заказано по конкретному ПЗ" - "Собранное в наборы кол-во сборки"`

#### Заказано по ПЗ {#zakazano-po-pz}

`Сумма всех "Потребность по ПЗ" на конкретную сущность`

#### Время по операциям ТП {#vremya-po-operatsiyam-tp}

`"Подготовительное время" + ("Вспомогательное время" + "Основное время") * Кол-во`

#### Дефицит с учетом производства  {#defitsit-s-uchetom-proizvodstva}
`'"Количество на складе" + "Количество деталей, заказанных на производство" - "Дефицит"'`

### 6. Расчет показателей операции {#f6-raschet-pokazatelei-operation}

#### Рассчет изготовление сущности на операцию {#calculate-prepare-entity-by-operation}
`("Подготовительное время" + ("Основное время" + "Вспомогательное") * "Кол-во сущности" )`

#### Рассчет изготовление сущности по ТП {#calculate-prepare-entity-by-tech-process}
`"Подготовительное время" + ("Основное время" + "Вспомогательное") по каждой операции * "Кол-во сущности")`

## Принципы и соглашения

- Формулы, помеченные как **«не может быть меньше нуля»**, должны возвращать 0, если результат вычисления отрицательный.
- Под **«Сумма[]»** подразумевается агрегация по всем родительским сущностям, где текущая сущность является компонентом.
- Расчеты, основанные на **«Потребности родителя»**, выполняются рекурсивно вверх по иерархии спецификаций.
